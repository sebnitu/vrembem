## Options

Drawer is built as a Sass module to handle styles and CSS output, along with a corresponding JavaScript class for managing behavior and plugin integration. Below are the available configuration options.

### JavaScript

Drawer comes with the following JavaScript configuration options that customize the behavior, accessibility, and integration of the drawer collection.

<CodeExpander lang="js">
```js
const drawer = new Drawer({
  // An object containing pre-configuration options for plugins. Object should
  // contain the name of the plugin to configure as the key and the config
  // object for that plugin as the value.
  // @type object
  presets,

  // The data attribute to use for triggering the open method
  // @type string
  attrOpen: "drawer-open",

  // The data attribute to use for triggering the close method
  // @type string
  attrClose: "drawer-close",

  // The data attribute to use for triggering the toggle method
  // @type string
  attrToggle: "drawer-toggle",

  // A valid CSS selector used to query the document for elements to include as
  // entries in the drawer collection.
  // @type string
  selector: ".drawer",

  // A valid CSS selector for the drawer dialog element
  // @type string
  selectorDialog: ".drawer__dialog",

  // A valid CSS selector for the element that should be treated as the modal
  // backdrop. Clicking the backdrop closes any open modal drawer.
  // @type string
  selectorBackdrop: ".drawer",

  // A valid CSS selector for setting the focus element of a drawer when opened
  // @type string
  selectorFocus: "[data-focus]",

  // A valid CSS selector for an element or group of elements that should be set
  // to `inert` when a modal drawer is opened.
  // @type string
  selectorInert: "",

  // A valid CSS selector for an element or group of elements that should have
  // overflow style set to "hidden" when a modal drawer is opened.
  // @type string
  selectorOverflow: "body",

  // A CSS class applied as the opened state of a drawer
  // @type string
  stateOpened: "is-opened",

  // A CSS class applied as the opening state of a drawer
  // @type string
  stateOpening: "is-opening",

  // A CSS class applied as the closing state of a drawer
  // @type string
  stateClosing: "is-closing",

  // A CSS class applied as the closed state of a drawer
  // @type string
  stateClosed: "is-closed",

  // A CSS class used to toggle the modal styles of a drawer
  // @type string
  classModal: "drawer_modal",

  // Whether or not to set the tabindex attribute of the drawer dialog
  // @type boolean
  setTabindex: true,

  // Whether or not to enable drawer transition states and animations
  // @type boolean
  transition: true,

  // The duration of transition animations
  // @type number (milliseconds)
  transitionDuration: 300
});
```
</CodeExpander>

### Sass

Drawer comes with the following Sass configuration options that can be set using either the [config module](/packages/core/config#using-config-module) or [Sass `with` clause](/packages/core/config#using-with-clause) methods.

<CodeExpander lang="scss">
```scss
@use "@vrembem/drawer" with (
  $config: (
    // A map containing drawer elements as keys and their preferred class 
    // selectors. These are layout elements used to provide the context styles for
    // drawers but are independent from the drawer component itself.
    // @type map
    "drawer-classes": (
      "frame": "drawer-frame",
      "main": "drawer-main"
    ),

    // Used to set the width property of drawers and margin clearance of the 
    // drawer-main element for inline drawers.
    // @type number
    "drawer-width": 18em,

    // Used to set the max-width property of drawers
    // @type percentage
    "drawer-max-width": 80%,

    // A map of CSS variable overrides to pass to the drawer dialog element
    // @type map
    "drawer-dialog-override": (),

    // A map of CSS variable overrides to pass to the drawer inline dialog element
    // @type map
    "drawer-inline-dialog-override": (
      "box-shadow": none
    ),

    // A map of CSS variable overrides to pass to the drawer modal dialog element
    // @type map
    "drawer-modal-dialog-override": (
      "border": none
    )
  )
);
```
</CodeExpander>

## Basic usage

Drawers are composed using classes and data attributes for their triggers. The basic structure of a drawer is an element with an `id` and `drawer` class, containing a child element with the `drawer__dialog` class. Two structural elements are required for drawers to work correctly:

- `drawer-frame`: Applied to the parent element wrapping all drawers and the main content.
- `drawer-main`: Applied to the element containing the main content. This should be the last child of the `drawer-frame` element.
- `drawer`: The base drawer element representing an individual drawer. In it's modal state this element acts as the drawer's backdrop.
- `drawer__dialog`: The content container of a drawer. Can be paired with the [dialog component](/package/dialog) for structured layouts with headers, bodies, and footers.
- `drawer__container`: Provides a scrollable container that separates content scrolling from the dialog's background and border styles, preventing overscroll visual issues.

<CodeExample flush>
  <Fragment slot="output">
    <div class="drawer-frame">
      <aside id="drawer-1" class="drawer is-closed">
        <div class="drawer__dialog">
          <div class="drawer__container padding">
            <div class="flex justify-content-between">
              <span>Drawer example</span>
              <button class="link" data-drawer-close="">Close</button>
            </div>
          </div>
        </div>
      </aside>
      <div class="drawer-main padding">
        <ul class="list">
          <li><button class="link" data-drawer-open="drawer-1">Open</button></li>
          <li><button class="link" data-drawer-close="drawer-1">Close</button></li>
          <li><button class="link" data-drawer-toggle="drawer-1">Toggle</button></li>
        </ul>
      </div>
    </div>
  </Fragment>
  ```html
  <!-- The drawer and content wrapper -->
  <div class="drawer-frame">

    <!-- The drawer element -->
    <aside id="[unique-id]" class="drawer">
      <div class="drawer__dialog">
        <!-- Optional container element for separating scrolling content -->
        <div class="drawer__container">
          ...
        </div>
      </div>
    </aside>

    <!-- The main content container -->
    <div class="drawer-main">
      ...
    </div>
  </div>
  ```
</CodeExample>

Drawer triggers are defined using three data attributes:

- `data-drawer-open`: Opens a drawer. Takes the ID of the drawer to open.
- `data-drawer-close`: Closes a drawer. It will close the parent drawer if left value-less but can also take the ID of a specific drawer to close.
- `data-drawer-toggle`: Toggles a drawer open or closed. It takes the ID of the drawer to toggle.

<CodeExample>
  ```html
  <button data-drawer-open="drawer-id">...</button>
  <button data-drawer-close="drawer-id">...</button>
  <button data-drawer-toggle="drawer-id">...</button>
  ```
</CodeExample>

The dialog element of a drawer is defined using the `drawer__dialog` class. Along with a `role` attribute (e.g., `role="dialog"`), authors should provide drawer dialogs with [`aria-labelledby`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Attributes/aria-labelledby) and [`aria-describedby`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Attributes/aria-describedby) attributes, if applicable, to further improve accessibility. The `aria-modal` attribute is applied automatically based on whether the drawer is currently in modal mode.

<CodeExample flush>

      <aside id="drawer-2" class="drawer is-closed">
        <div class="drawer__dialog dialog" role="dialog" aria-labelledby="dialog-title" aria-describedby="dialog-description">
          <div class="dialog__header justify-content-between">
            <h2 id="dialog-title" class="dialog__title">Drawer title</h2>
            <button class="link" data-drawer-close="">Close</button>
          </div>
          <div class="dialog__body">
            <p id="dialog-description">This is a drawer example using the dialog component.</p>
          </div>
          <div class="dialog__footer">
            Drawer footer.
          </div>
        </div>
      </aside>
      <div class="drawer-main padding">
        <ul class="list">
          <li><button class="link" data-drawer-open="drawer-2">Open</button></li>
          <li><button class="link" data-drawer-close="drawer-2">Close</button></li>
          <li><button class="link" data-drawer-toggle="drawer-2">Toggle</button></li>
        </ul>
      </div>
    </div>
  </Fragment>
  ```html
  <aside id="[unique-id]" class="drawer">
    <div class="drawer__dialog dialog" role="dialog" aria-labelledby="dialog-title" aria-describedby="dialog-description">
      <div class="dialog__header">
  ```

The [dialog component](/package/dialog) is a great fit for composing a drawer's dialog. This can be applied directly on the `drawer__dialog` element and expanded as needed.

### Modal drawers

There are multiple ways a modal drawer can be created. The simplest is by applying the [`drawer_modal`](#drawer_modal) modifier class to the `drawer` element before it has been registered. To change a drawer's mode after it has been registered, change the `mode` property of the drawer entry from `"inline"` to `"modal"` (or vice versa) in JavaScript.

<CodeExample tabs flush>
  <Fragment slot="output">
    <div class="drawer-frame" style="min-height: 8rem;">
      <aside id="drawer-3" class="drawer drawer_modal" data-config="{ 'selectorInert': null, 'selectorOverflow': null }">
        <div class="drawer__dialog dialog">
          <div class="dialog__body">
            <div class="flex">
              <h2 class="flex-grow">Modal drawer</h2>
              <button class="link" data-drawer-close="">Close</button>
            </div>
          </div>
        </div>
      </aside>
      <div class="drawer-main padding">
        <button class="link" data-drawer-toggle="drawer-3">Toggle</button>
      </div>
    </div>
  </Fragment>
  <Fragment slot="tab-html">
    ```html
    <aside id="[unique-id]" class="drawer drawer_modal">
      ...
    </aside>
    ```
  </Fragment>
  <Fragment slot="tab-js">
    ```js
    // Set a drawer to "modal" mode
    drawer.get("drawer-id").mode = "modal";

    // Set a drawer to "Inline" mode
    drawer.get("drawer-id").mode = "inline";
    ```
  </Fragment>
</CodeExample>

In cases where you'd like a drawer to switch modes based on a specific viewport width, use the [`mediaQuery`](/packages/core/plugins/media-query) plugin. Set a `data-breakpoint` data attribute with either a width value or a breakpoint key.

            <code>data-breakpoint="md"</code>

            <button class="link" data-drawer-toggle="drawer-5">Toggle</button> - Switches to modal below `"md"` breakpoint viewports.


To help improve accessibility, use the [`focusTrap`](/packages/core/plugins/focus-trap) plugin when creating modal drawers. While a modal drawer is active, the content obscured by the modal is made inaccessible to all users via a focus trap. This prevents the `TAB` key and screen reader's virtual cursor (arrow keys) from leaving the drawer dialog.

### Focus management

Drawer dialogs are given focus when opened as long as the `setTabindex` option is set to `true` or if the drawer dialog has `tabindex="-1"` set manually. If focus on a specific element inside a drawer is preferred, give that element the `data-focus` attribute. Focus is returned to the element that initially triggered the drawer once it is closed.

<CodeExample flush>
  <Fragment slot="output">
    <div class="drawer-frame" style="min-height: 8rem;">
      <aside id="drawer-6" class="drawer is-closed">
        <div class="drawer__dialog dialog">
          <div class="dialog__header">
            <h2 class="dialog__title">Drawer example</h2>
            <button class="link" data-drawer-close="">Close</button>
          </div>
          <div class="dialog__body">
            <p>Focus should be initially set on the dialog element.</p>
          </div>
        </div>
      </aside>
      <aside id="drawer-7" class="drawer is-closed">
        <div class="drawer__dialog dialog">
          <div class="dialog__header">
            <h2 class="dialog__title">Drawer example</h2>
            <button class="link" data-drawer-close="">Close</button>
          </div>
          <div class="dialog__body">
            <input class="input" data-focus type="text" />
          </div>
        </div>
      </aside>
      <div class="drawer-main padding">
        <ul class="list">
          <li>
            <button class="link" data-drawer-toggle="drawer-6">Toggle</button> - Sets focus to the drawer dialog when opened.
          </li>
          <li>
            <button class="link" data-drawer-toggle="drawer-7">Toggle</button> - Sets focus to the `data-focus` element when opened.
          </li>
        </ul>
      </div>
    </div>
  </Fragment>
  ```html
  <div class="drawer-frame">
    <!-- Focuses the drawer dialog on open -->
    <aside id="[unique-id]" class="drawer">
      <div class="drawer__dialog" tabindex="-1">
        ...
      </div>
    </aside>
    <!-- Focuses an inner element on open -->
    <aside id="[unique-id]" class="drawer">
      <div class="drawer__dialog">
        <input data-focus type="text" />
      </div>
    </aside>
    
    <div class="drawer-main">
      <!-- Return focus to toggle on close -->
      <button data-drawer-toggle="[unique-id]">...</button>
    </div>
  </div>
  ```
</CodeExample>

To change the selector used in finding the preferred focus element, pass your own selector via the `selectorFocus` config option (defaults to `'[data-focus]'`).

### Drawer state

Drawer state is managed using three properties on drawer entries:

- [`entry.mode`](#entrymode): Stores and allows changing the current mode of the drawer.
- [`entry.state`](#entrystate): Stores the current state of the drawer.
- [`entry.inlineState`](#entryinlinestate): Stores the inline state of the drawer. This is used to restore the inline state when a drawer's mode is switched from `modal` to `inline`.

To persist inline drawer state across browser page reloads, use the [`propStore`](/packages/core/plugins/prop-store) plugin.

### Behavior and accessibility

Drawers in their modal state follow a set of patterns expected from other modals on the web. Here's what to expect:

1. When a modal drawer is opened, focus is moved to the dialog or an element inside.
2. Modal drawers provide standard methods for the user to close, such as using the `esc` key or clicking outside the dialog.
3. While the modal drawer is active, content obscured by the modal is inaccessible to all users.
4. When a modal drawer is closed, focus is returned to the initial trigger element that activated it.

To take full advantage of the modal drawer's accessibility features, it's recommended to set the `selectorInert` option to all elements that are outside the modal (most likely the `drawer-main` element). All elements that match the `selectorInert` selector will be given the `inert` attribute, as well as `aria-hidden="true"`, when a modal is opened.

#### Example

Here's an example where we want the `drawer-main` content area to be inaccessible while drawer modals are open. We also want to disable other scrollable elements using the `selectorOverflow` option.

<CodeExample lang="js">

   ```js
   const drawer = new Drawer({
     selectorInert: ".drawer-main",
     selectorOverflow: "body, .drawer-main"
   });

   await drawer.mount();
   ```
</CodeExample>

## Reference

`Drawer` and `DrawerEntry` extend the [`Collection`](/packages/core/collection-api) and [`CollectionEntry`](/packages/core/collection-entry-api) classes, respectively, and inherit their APIs. This reference contains only drawer-specific API properties and methods. Check out the [collections documentation](/packages/core#collections) for more information on inherited APIs.

### activeModal

<ReferenceBar type="DrawerEntry | undefined" />

Returns the active modal drawer if one exists. Otherwise, returns undefined.

#### Example

<CodeExample lang="js">
```js
if (drawer.activeModal) {
  // This block will run if a modal drawer is active...
}
```
</CodeExample>

### open

<ReferenceBar type="method" returns="Promise<DrawerEntry>" />

Opens a drawer using the provided drawer ID. It will throw an error if the provided ID does not match an existing drawer entry.

#### Arguments

<ReferenceCard name="id" type="string">
  The ID of the drawer to open.
</ReferenceCard>

<ReferenceCard name="transition" type="boolean" defaults="true">
  Whether or not to animate transition between states. The transitional state class used is `is-opening`.
</ReferenceCard>

<ReferenceCard name="focus" type="boolean" defaults="true">
  Whether or not to update the focus state after the drawer is opened. When true, focus is set to any element inside the drawer with the `data-focus` attribute else the drawer dialog element.
</ReferenceCard>

#### Example

<CodeExample lang="js">
```js
await drawer.open("my-drawer");
// > Opens the drawer with id="my-drawer"

await drawer.open("fake-drawer");
// > Uncaught Error: Drawer entry not found in collection with id of "fake-drawer"
```
</CodeExample>

### close

<ReferenceBar type="method" returns="Promise<DrawerEntry>" />

Closes a drawer using the provided drawer ID. It will throw an error if the provided ID does not match an existing drawer entry.

#### Arguments

<ReferenceCard name="id" type="string">
  The ID of the drawer to close.
</ReferenceCard>

<ReferenceCard name="transition" type="boolean" defaults="true">
  Whether or not to animate transition between states. The transitional state class used is `is-closing`.
</ReferenceCard>

<ReferenceCard name="focus" type="boolean" defaults="true">
  Whether or not to update the focus state after the drawer is closed. When true, focus is set to the stored element (if any) that triggered the drawer to open.
</ReferenceCard>

#### Example

<CodeExample lang="js">
```js
await drawer.close("my-drawer");
// > Closes the drawer with id="my-drawer"

await drawer.close("fake-drawer");
// > Uncaught Error: Drawer entry not found in collection with id of "fake-drawer"
```
</CodeExample>

### toggle

<ReferenceBar type="method" returns="Promise<DrawerEntry>" />

Toggles a drawer open or closed depending on its current state using the provided drawer ID. It will throw an error if the provided ID does not match an existing drawer entry.

#### Arguments

<ReferenceCard name="id" type="string">
  The ID of the drawer to toggle.
</ReferenceCard>

<ReferenceCard name="transition" type="boolean" defaults="true">
  Whether or not to animate transition between states. The transitional state class used is either `is-opening` or `is-closing`.
</ReferenceCard>

<ReferenceCard name="focus" type="boolean" defaults="true">
  Whether or not to update the focus state after the drawer is toggled. When true, focus is handled appropriately depending on whether the drawer is being opened or closed.
</ReferenceCard>

#### Example

<CodeExample lang="js">
```js
await drawer.toggle("my-drawer");
// > Toggles the drawer with id="my-drawer"

await drawer.toggle("fake-drawer");
// > Uncaught Error: Drawer entry not found in collection with id of "fake-drawer"
```
</CodeExample>

### entry.mode

<ReferenceBar type="string" />

Stores and allows changing the current mode of the drawer. There are two modes a drawer can be in:
- `inline`: This mode is set within the context of the page, typically rendered as an aside. Though it can still be toggled, when opened, it does not restrict access to other content.
- `modal`: A modal drawer is initially hidden and is rendered on top of other page elements when opened.

A drawer's mode can be changed at any time but is typically toggled based on a viewport breakpoint using the [`mediaQuery`](/packages/core/plugins/media-query) plugin.

#### Example

<CodeExample lang="js">
```js
// Get the drawer element
const entry = drawer.get("my-drawer");

// Switch the drawer to inline
entry.mode = "inline";

// Switch the drawer to modal
entry.mode = "modal";

// Will throw an error if set to an invalid mode
entry.mode = "asdf";
// > Uncaught Error: Not a valid drawer mode: "asdf"
```
</CodeExample>

### entry.state

<ReferenceBar type="string" />

Stores the current state of the drawer. Possible states include:

- `opened`: The drawer is currently open, and its content is accessible.
- `opening`: A transitional state from closed to open.
- `closed`: The drawer is currently closed, and its content is hidden.
- `closing`: A transitional state from open to closed.
- `indeterminate`: A state that is not directly managed by JavaScript or cannot be determined. This is typically applied for inline drawers whose visibility is managed by CSS instead of JavaScript.

### entry.inlineState

<ReferenceBar type="string" />

Stores the inline state of the drawer. This is used to restore the inline state when a drawer's mode is switched from `modal` to `inline`. The inline drawer state can be persisted across page loads using the [`propStore`](/packages/core/plugins/prop-store) plugin. Possible states include:

- `opened`: The drawer is currently open, and its content is accessible.
- `closed`: The drawer is currently closed, and its content is hidden.
- `indeterminate`: A state that is not directly managed by JavaScript or cannot be determined. This is typically applied for inline drawers whose visibility is managed by CSS instead of JavaScript.

### entry.dialog

<ReferenceBar type="HTMLEntry" />

Stores a reference to the dialog element (`.drawer__dialog`) of the drawer entry. If a dialog element isn't found, this property points to the drawer element (`.drawer`) itself.

### entry.trigger

<ReferenceBar type="HTMLEntry | null" />

Stores a reference to a trigger element. When a drawer is opened, the trigger (typically a button) is the element that was used to open it. Focus is returned to this element when the drawer is closed.

### entry.open

<ReferenceBar type="method" returns="Promise<DrawerEntry>" />

Opens the drawer if it is currently closed or indeterminate.

#### Arguments

<ReferenceCard name="transition" type="boolean" defaults="true">
  Whether or not to animate transition between states. The transitional state class used is `is-opening`.
</ReferenceCard>

<ReferenceCard name="focus" type="boolean" defaults="true">
  Whether or not to update the focus state after the drawer is opened. When true, focus is set to any element inside the drawer with the `data-focus` attribute else the drawer dialog element.
</ReferenceCard>

#### Example

<CodeExample lang="js">
```js
// Get the drawer element
const entry = drawer.get("my-drawer");

// Open the drawer
await entry.open();
```
</CodeExample>

### entry.close

<ReferenceBar type="method" returns="Promise<DrawerEntry>" />

Closes the drawer if it is currently open or indeterminate.

#### Arguments

<ReferenceCard name="transition" type="boolean" defaults="true">
  Whether or not to animate transition between states. The transitional state class used is `is-closing`.
</ReferenceCard>

<ReferenceCard name="focus" type="boolean" defaults="true">
  Whether or not to update the focus state after the drawer is closed. When true, focus is set to the stored element (if any) that triggered the drawer to open.
</ReferenceCard>

#### Example

<CodeExample lang="js">
```js
// Get the drawer element
const entry = drawer.get("my-drawer");

// Close the drawer
await entry.close();
```
</CodeExample>

### entry.toggle

<ReferenceBar type="method" returns="Promise<DrawerEntry>" />

Toggles the drawer open or closed depending on its current state.

#### Arguments

<ReferenceCard name="transition" type="boolean" defaults="true">
  Whether or not to animate transition between states. The transitional state class used is either `is-opening` or `is-closing`.
</ReferenceCard>

<ReferenceCard name="focus" type="boolean" defaults="true">
  Whether or not to update the focus state after the drawer is toggled. When true, focus is handled appropriately depending on whether the drawer is being opened or closed.
</ReferenceCard>

#### Example

<CodeExample lang="js">
```js
// Get the drawer element
const entry = drawer.get("my-drawer");

// Toggle the drawer
await entry.toggle();
```
</CodeExample>

### Lifecycle Hooks

In addition to the inherited [lifecycle hooks](./core/collections#lifecycle-hooks), Drawer provides three of its own events to hook into its lifecycle:

<ReferenceCard name="opened" type="event">
  Emitted when a drawer is opened.

  ```js
  drawer.on("opened", callback({ parent, entry }));
  ```
</ReferenceCard>

<ReferenceCard name="closed" type="event">
  Emitted when a drawer is closed.

  ```js
  drawer.on("closed", callback({ parent, entry }));
  ```
</ReferenceCard>

<ReferenceCard name="switchMode" type="event">
  Emitted when a drawer switches modes.

  ```js
  drawer.on("switchMode", callback({ parent, entry }));
  ```
</ReferenceCard>
