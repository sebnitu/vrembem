## Options

Button is built as a Sass module. Sass options are used to control CSS output, while CSS variables control the styling of the component. Below are the available configuration options.

### Sass

Button comes with the following Sass configuration options that can be set using either the [config module](/packages/core/config#using-config-module) or [Sass `with` clause](/packages/core/config#using-with-clause) methods.

<CodeExpander lang="scss">
```scss
@use "@vrembem/button" with (
  $config: (
    // Toggle for outputting button_block modifier breakpoint variants
    // @type boolean
    "button-block-breakpoints": true,

    // Map for outputting button_color_[key] modifier variants
    // @type map
    "button-color": (
      "primary": palette.get("primary"),
      "secondary": palette.get("secondary"),
      "neutral": palette.get("neutral"),
      "important": palette.get("important")
    ),

    // Map for outputting button_size_[key] modifier variants
    // @type map
    "button-size": (
      "breakpoints": true,
      "sm": css.get("form-control-size-sm"),
      "md": css.get("form-control-size"),
      "lg": css.get("form-control-size-lg")
    )
  )
);
```
</CodeExpander>

## Basic usage

The most basic implementation of the button component consists of the `button` class that can be applied to `<button>`, `<a role="button">` or `<input>` elements with a type attribute of "button", "submit" or "reset". [Remember to add `role="button"`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/button_role) to any none semantic buttons as needed.

<CodeExample>
  <div slot="output" class="flex">
    <button class="button">Button</button>
    <a class="button" role="button">Anchor</a>
    <input class="button" type="button" value="Input" />
    <input class="button" type="submit" value="Submit" />
    <input class="button" type="reset" value="Reset" />
  </div>
  ```html
  <button class="button">Button</button>
  <a class="button" role="button">Anchor</a>
  <input class="button" type="button" value="Input" />
  <input class="button" type="submit" value="Submit" />
  <input class="button" type="reset" value="Reset" />
  ```
</CodeExample>

Elements within a button are centered vertically and spaced accordingly using the value set by `--vb-button-gap` (default: `0.5em`). When elements are added inside a button, the button's text should also be wrapped in a `<span>` so that it can be spaced properly.

<CodeExample>
  <Fragment slot="output">
    <button class="button">
      <Icon name="anchor" classes="icon_size_sm" />
      <span>Button</span>
      <span class="arrow"></span>
    </button>
  </Fragment>
  ```html
  <button class="button">
    <svg class="icon icon_size_sm">...</svg>
    <span>Button</span>
    <span class="arrow"></span>
  </button>
  ```
</CodeExample>

### Disabled

When disabled using the `disabled` attribute, a button will inherit styles to visually appear noninteractive.

<CodeExample>
  <Fragment slot="output">
    <button class="button" disabled>Button</button>
  </Fragment>
  ```html
  <button class="button" disabled>Button</button>
  ```
</CodeExample>

### Loading

Buttons can also be given a loading state by adding the `is-loading` state class. This is useful when a button has an action that has a delayed response in order to give users a visual indicator that their action is being processed.

<CodeExample>
  <div slot="output">
    <button class="button is-loading" disabled>Button</button>
  </div>
  ```html
  <button class="button is-loading" disabled>Button</button>
  ```
</CodeExample>

Keep in mind that the `is-loading` state class will not work on `<input type="button">` since it uses the `::after` pseudo-element to display the loading spinner.

## button_block

Modifier that gives buttons a "block" appearance so that it spans the full width of its container.

<CodeExample>
  <Fragment slot="output">
    <button class="button button_block">Button</button>
  </Fragment>
  ```html
  <button class="button button_block">Button</button>
  ```
</CodeExample>

### Breakpoint variants

These modifiers come with media breakpoint variants. This allows changing styles based on a specific breakpoint key in the [`breakpoints`](/packages/core#config-options) config options map. Breakpoint variants output can be toggled using the `button-block-breakpoints` config option (default: `true`).

<CodeExample>
  <Fragment slot="demo">
    <label class="text-nowrap" for="demo-button-block-select">Variants Demo:</label>
    <select id="demo-button-block-select" class="input input_type_select input_size_sm input_auto" data-demo-target="demo-button-block">
      <option value="xs:button_block">xs:button_block</option>
      <option value="sm:button_block">sm:button_block</option>
      <option value="md:button_block">md:button_block</option>
      <option value="lg:button_block">lg:button_block</option>
      <option value="xl:button_block">xl:button_block</option>
    </select>
  </Fragment>
  <div slot="output" class="spacing">
    <p>Current view-port width: <code data-vp-width>...</code></p>
    <button id="demo-button-block" class="button lg:button_block">Button</button>
  </div>
  ```html
  <button class="button lg:button_block">Button</button>
  ```
</CodeExample>

**Available variants**

- `xs:button_block`
- `sm:button_block`
- `md:button_block`
- `lg:button_block`
- `xl:button_block`

## button_color_[key]

Modifiers for changing the look and feel of a button. These are usually done with different background and text colors. These modifiers are generated using key/value pairs of the `button-color` config options map. The values set in `button-color` are passed directly to the [`button-variant`](#button-variant) mixin.

<CodeExample>
  <div slot="output" class="flex">
    <button class="button button_color_primary">Button</button>
    <button class="button button_color_secondary">Button</button>
    <button class="button button_color_neutral">Button</button>
    <button class="button button_color_important">Button</button>
  </div>
  ```html
  <button class="button button_color_primary">Button</button>
  <button class="button button_color_secondary">Button</button>
  <button class="button button_color_neutral">Button</button>
  <button class="button button_color_important">Button</button>
  ```
</CodeExample>

**Available variants**

- `button_color_primary`
- `button_color_secondary`
- `button_color_neutral`
- `button_color_important`

## button_icon

Adds styles that make icon-only buttons have equal padding. This will result in a square button if the icon used also has an equal width and height.

<CodeExample>
  <div slot="output">
    <button class="button button_icon">
      <Icon name="anchor" />
    </button>
  </div>
  ```html
  <button class="button button_icon">
    <svg class="icon">...</svg>
  </button>
  ```
</CodeExample>

## button_size_[key]

Adjust the size of a button by increasing or decreasing its padding and font-size. These modifiers are generated using key/value pairs of the `button-size` config options map.

<CodeExample>
  <div slot="output" class="flex align-items-center">
    <button class="button button_size_sm">Button</button>
    <button class="button button_size_md">Button</button>
    <button class="button button_size_lg">Button</button>
  </div>
  ```html
  <button class="button button_size_sm">Button</button>
  <button class="button button_size_md">Button</button>
  <button class="button button_size_lg">Button</button>
  ```
</CodeExample>

**Available variants**

- `button_size_sm`
- `button_size_md`
- `button_size_lg`

### Breakpoint variants

These modifiers come with media breakpoint variants. This allows changing styles based on a specific breakpoint key in the [`breakpoints`](/packages/core#config-options) config options map. Breakpoint variants output can be toggled using the `breakpoints` key in the `button-size` config options map (default: `true`).

<CodeExample>
  <div slot="output" class="spacing">
    <p>Current view-port width: <code data-vp-width>...</code></p>
    <button class="button button_size_sm lg:button_size_md xl:button_size_lg">
      <span>Button</span>
      <span class="badge lg:display-none">size_sm</span>
      <span class="badge display-none lg:display-block xl:display-none">size_md</span>
      <span class="badge display-none xl:display-block">size_lg</span>
    </button>
  </div>
  ```html
  <button class="button button_size_sm lg:button_size_md xl:button_size_lg">
    Button
  </button>
  ```
</CodeExample>

**Available variants**

- `xs:button_size_[key]`
- `sm:button_size_[key]`
- `md:button_size_[key]`
- `lg:button_size_[key]`
- `xl:button_size_[key]`

## Reference

Below is a reference of all available button mixins.

### button.base

<ReferenceBar type="mixin" />

Output the base styles for a button component. This includes:
- Setup of button specific CSS variables with shared form-control fallbacks.
- Setup of focus-ring CSS variables and initial state.
- Flex layout styles for child elements and consistent gap spacing.

<CodeExample lang="scss">
```scss
@use "@vrembem/button";

.custom-button {
  @include button.base;
  // Your custom variant styles here...
}
```
</CodeExample>

### button.variant

<ReferenceBar type="mixin" />

Creates a button variant using custom properties and hsl relative values.

#### Arguments

<ReferenceCard name="$background" type="color | map">
  The color to use as the button background. Also accepts a map that defines multiple override properties.
</ReferenceCard>

<ReferenceCard name="$foreground" type="color" defaults="white">
  The color to use as the foreground override.
</ReferenceCard>

<ReferenceCard name="$operator" type="string" defaults='"-"'>
  The operator to use on the lightness value. Can either be `"-"` to darken or `"+"` to lighten the base lightness value.
</ReferenceCard>

#### Example

<CodeExample tabs>
<Fragment slot="output">
  <button class="button button_color_custom">Custom Button</button>
</Fragment>
<Fragment slot="tab-scss">
```scss
@use "@vrembem/button";

.button_color_custom {
  @include button.variant(salmon);
}
```
</Fragment>
<Fragment slot="tab-css">
```css
.button_color_custom {
  --vb-button-background: salmon;
  --vb-button-foreground: white;
  --vb-button-border-color: transparent;
  --vb-button-box-shadow: none;
  --vb-button-accent: salmon;
  --vb-button-background-hover: hsl(from salmon h s calc(l - 5));
  --vb-button-background-active: hsl(from salmon h s calc(l - 10));
}
```
</Fragment>
</CodeExample>
