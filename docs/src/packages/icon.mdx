## Options

Icon is built as a Sass module. Sass options are used to control CSS output, while CSS variables control the styling of the component. Below are the available configuration options.

### Sass

Icon comes with the following Sass configuration options that can be set using either the [config module](/packages/core/config#using-config-module) or [Sass `with` clause](/packages/core/config#using-with-clause) methods.

<CodeExpander lang="scss">
```scss
@use "@vrembem/icon" with (
  $config: (
    // Map for outputting the icon_style_[key] modifier variants
    // @type map
    "icon-style": (
      "default": "stroke",
      "stroke": "stroke",
      "fill": "fill",
      "both": "both"
    ),

    // Map for outputting the icon_size_[key] modifier variants. Values should 
    // contain a list of font-size and stroke-width pairs.
    // @type map
    "icon-size": (
      "default": 1.5em 2,
      "xs": 0.75em 3.5,
      "sm": 1.125em 2.5,
      "lg": 2em 1.75,
      "xl": 3em 1.3
    )
  )
);
```
</CodeExpander>

### CSS Variables

Icon uses CSS variables in a reference and fallback pattern. Variables are referenced in the component's styles but are not directly defined; instead, they are provided with sensible fallbacks.

  "--vb-icon-size": "1.5em",
  "--vb-icon-stroke-width": "2",
}} />

## Basic usage

The most basic implementation of the icon component consists of the `icon` class applied to an svg element.

<CodeExample>
  <div slot="output" class="flex flex_gap_xl">
    <Icon name="anchor" />
    <Icon name="arrow-left" />
    <Icon name="arrow-right" />
    <Icon name="arrow-up" />
    <Icon name="arrow-down" />
    <Icon name="clipboard" />
    <Icon name="clock" />
    <Icon name="download-cloud" />
  </div>
  ```html
  <svg class="icon" role="img">...</svg>
  ```
</CodeExample>

## icon_size_[key]

Add a size modifier to adjust the size of your icons. These modifiers are generated using key/value pairs of the `icon-size` config options map.

<CodeExample>
  <div slot="output" class="flex flex_gap_xl">
    <Icon name="anchor" classes="icon_size_xs" />
    <Icon name="anchor" classes="icon_size_sm" />
    <Icon name="anchor" />
    <Icon name="anchor" classes="icon_size_lg" />
    <Icon name="anchor" classes="icon_size_xl" />
  </div>
  ```html
  <svg class="icon icon_size_sm" role="img">...</svg>
  ```
</CodeExample>

**Available variants**

- `icon_size_xs`
- `icon_size_sm`
- `icon_size_lg`
- `icon_size_xl`

## icon_style_[key]

Adjust the icon style to accommodate stroke, filled or stroke and filled style icons. These modifiers are generated using key/value pairs of the `icon-style` config options map.

<CodeExample>
  <div slot="output" class="flex flex_gap_xl">
    <Icon name="heart" classes="icon_style_stroke" />
    <Icon name="heart" classes="icon_style_fill" />
    <Icon name="heart" classes="icon_style_both" />
    <Icon name="sun" classes="icon_style_stroke" />
    <Icon name="sun" classes="icon_style_fill" />
    <Icon name="sun" classes="icon_style_both" />
  </div>
  ```html
  <svg class="icon icon_style_stroke" role="img">...</svg>
  <svg class="icon icon_style_fill" role="img">...</svg>
  <svg class="icon icon_style_both" role="img">...</svg>
  ```
</CodeExample>

**Available variants**

- `icon_style_stroke`
- `icon_style_fill`
- `icon_style_both`

## Reference

Below is a reference of all available icon mixins.

### icon.style

<ReferenceBar type="mixin" />

Outputs the required properties for a specific icon style.

#### Arguments

<ReferenceCard name="$style" type="string">
  The name of the styles to apply. Can be either `"stroke"`, `"fill"` or `"both"`.
</ReferenceCard>

#### Example

<CodeExample lang="scss">
```scss
@use "@vrembem/icon";

.custom-icon {
  @include icon.style("stroke");
  // Your custom variant styles here...
}
```
</CodeExample>
