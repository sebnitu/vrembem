## Using CDN

For fast prototyping, use your favorite CDN (Content Delivery Network) to get Vrembem into your project without needing to install and build from NPM.

<CodeExample>
```html
<link rel="stylesheet" href="https://unpkg.com/vrembem@next/dist/index.css" />
```
</CodeExample>

{/*

- `https://unpkg.com/vrembem@next/dist/index.js`
- `https://cdn.jsdelivr.net/npm/vrembem@next/dist/index.css`
- `https://cdn.jsdelivr.net/npm/vrembem@next/dist/index.js`
*/}

To use available JavaScript modules such as [drawer](/packages/drawer), [modal](/packages/modal), [popover](/packages/popover), or plugins, import them directly from the `vrembem` package using the ESM bundle.

<CodeExample>
```html
<script type="module">
  import { ModalCollection, focusTrap } from "https://unpkg.com/vrembem@next/dist/index.js";
  
  const modals = new ModalCollection({
    plugins: [focusTrap()]
  });

  await modals.mount();
</script>
```
</CodeExample>

## Using NPM

Vrembem is a collection of packages built with Sass and TypeScript that compile to CSS and JavaScript. Install Vrembem via NPM and integrate it into your build pipeline to leverage Sass configurations, modules, and tree-shaking.

```sh
npm install vrembem
```

<CodeExample tabs>
  <Fragment slot="tab-scss">
    ```scss
    @use "vrembem";
    ```
  </Fragment>
  <Fragment slot="tab-css">
    ```css
    @import "vrembem";
    ```
  </Fragment>
</CodeExample>

<CodeExample lang="js">
```js
// Namespace import — access everything via a single object
import * as vb from "vrembem";

// Named imports — only pull in what you need (tree-shakable)
import { drawerCollection, propStore } from "vrembem";
```
</CodeExample>

### Entries

Vrembem provides both Sass (SCSS) and CSS entries. Which one to choose depends on project setup and how much control you need over the output.

<div class="compare">
  <div>
    <h4>SCSS</h4>
    <ul class="list list_pros">
      <li>Fine-grained control over output including prefixes and BEM naming conventions.</li>
      <li>Use Sass modules for managing configurations, design tokens, palettes, and theming.</li>
      <li>Best for projects already using Sass that want deeper integration into Vrembem systems.</li>
    </ul>
  </div>
  <span class="compare__vs"></span>
  <div>
    <h4>CSS</h4>
    <ul class="list list_pros">
      <li>Minimal setup and no build tools required.</li>
      <li>Customizable components using design token CSS variables.</li>
      <li>Great for projects that don't use Sass or want minimal dependencies.</li>
    </ul>
  </div>
</div>

<CodeExample tabs>
  <Fragment slot="tab-scss">
    ```scss
    // The entire Vrembem library
    @use "vrembem";

    // Component and utility styles
    @use "vrembem/styles";

    // CSS layer order
    @use "vrembem/layers";

    // Design token CSS variables
    @use "vrembem/tokens";
    ```
  </Fragment>
  <Fragment slot="tab-css">
    ```css
    /* The entire Vrembem library */
    @import "vrembem";

    /* Component and utility styles */
    @import "vrembem/styles";

    /* CSS layer order */
    @import "vrembem/layers";

    /* Design token CSS variables */
    @import "vrembem/tokens";
    ```
  </Fragment>
</CodeExample>

Components can be imported on a per-package basis using the `@vrembem/<package>` format. See [packages documentation](/packages) for more information on configuration and usage.
