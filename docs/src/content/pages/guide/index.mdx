Vrembem is a framework agnostic CSS library. Getting started only requires including a single CSS file into your project. If only a few components are needed, import them directly without the extra bloat.

### Starter template

Use this minimal starter template to begin prototyping with Vrembem now. Just copy/paste the contents into an HTML file locally and open it with your browser.

<CodeExample>
```html
<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Vrembem - Minimal starter template</title>
    <link rel="stylesheet" href="https://unpkg.com/vrembem@next/dist/index.css" />
  </head>
  <body>
    <section class="section">
      <div class="section__container">
        <div class="type spacing text-align-center">
          <h1>Hello World</h1>
          <p>A minimal starter template powered by Vrembem via CDN.</p>
          <div class="flex justify-content-center foreground-subtle">
            <a class="link" href="https://vrembem.com/">Docs</a>
            <span>/</span>
            <a class="link" href="https://github.com/sebnitu/vrembem">GitHub</a>
          </div>
        </div>
      </div>
    </section>
  </body>
</html>
```
</CodeExample>

Three components ([drawer](/packages/drawer), [modal](/packages/modal) and [popover](/packages/popover)) also ship with JavaScript which is built to be framework agnostic and tree-shakable. These can be included as needed, but are not required if you'd prefer to ship your own JavaScript integrations.

Get access to all provided JavaScript modules and plugins by importing them directly from the `vrembem` package or load them using a namespace import for quick prototyping.

<CodeExample>
```html
<script type="module">
  import * as vb from "https://unpkg.com/vrembem@next/dist/index.js";
  
  const modals = new vb.ModalCollection({
    plugins: [vb.focusTrap()]
  });

  await modals.mount();
</script>
```
</CodeExample>

## Using NPM

Built on Sass and TypeScript, Vrembem compiles to CSS and JavaScript modules. These can be imported on a per-package basis `@vrembem/<package>` or the entire library can be imported using the `vrembem` all-in-one package.

But when should you use direct CSS imports, or Sass modules? That mainly depends on your use case and how deeply you'd like to integrate into the Vrembem systems. Here is a list to help you decide:
