## Options

Input is built as a Sass module. Sass options are used to control CSS output, while design token CSS variables control the styling of the component. Below are the available configuration options.

### Sass

Input comes with the following Sass configuration options that can be set using either the [config module](/reference/sass-modules/config#using-config-module) or [Sass `with` clause](/reference/sass-modules/config#using-with-clause) methods.

<CodeExample lang="scss">
```scss
@use "@vrembem/input" with (
  $config: (
    // Map for outputting input_state_[key] modifier variants
    // @type map
    "input-state": (
      "success": palette.get("primary"),
      "error": palette.get("important")
    )
  )
);
```
</CodeExample>

## Basic usage

The most basic implementation of the input component consists of the `input` class applied to an `<input>` element.

<CodeExample>
  <Fragment slot="output">
    <input class="input" type="text" placeholder="Example input..." />
  </Fragment>
  ```html
  <input class="input" type="text" placeholder="Example input..." />
  ```
</CodeExample>

### Disabled

Adding the boolean `disabled` attribute to the input will provide visual indication that the input is not available for use.

<CodeExample>
  <Fragment slot="output">
    <input class="input" type="text" disabled />
  </Fragment>
  ```html
  <input class="input" type="text" disabled />
  ```
</CodeExample>

### Readonly

Adding the boolean `readonly` attribute to the input will provide visual indication that the user should not be able to edit the value of the input.

<CodeExample>
  <Fragment slot="output">
    <input class="input" type="text" value="Readonly example" readonly />
  </Fragment>
  ```html
  <input class="input" type="text" readonly />
  ```
</CodeExample>

## input_state_[key]

Apply styles for changing the look and feel of an input to better reflect the current state. These modifiers are generated using key/value pairs of the `input-state` config options map.

<CodeExample>
  <div slot="output" class="spacing">
    <input class="input input_state_success" type="text" />
    <input class="input input_state_error" type="text" />
  </div>
  ```html
  <input class="input input_state_success" type="text" />
  <input class="input input_state_error" type="text" />
  ```
</CodeExample>

**Available variants**

- `input_state_success`
- `input_state_error`

## Reference

Below is a reference of all available input mixins.

### input.base

<ReferenceBar type="mixin" />

Output the base styles for an input component. This includes:
- Setup of input specific CSS variables with shared form-control fallbacks.
- Setup of focus-ring CSS variables and initial state.
- Apply base styles and resets so browsers render inputs consistently.

<CodeExample lang="scss">
```scss
@use "@vrembem/input";

.custom-input {
  @include input.base;
  // Your custom variant styles here...
}
```
</CodeExample>
