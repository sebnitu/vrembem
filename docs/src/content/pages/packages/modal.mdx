## Options

Modal is built as a Sass module. Sass options are used to control CSS output, while design token CSS variables control the styling of the component. Below are the available configuration options.

### Sass

Modal comes with the following Sass configuration options that can be set using either the [config module](/reference/sass-modules/config#using-config-module) or [Sass `with` clause](/reference/sass-modules/config#using-with-clause) methods.

<CodeExample lang="scss">
```scss
@use "@vrembem/modal" with (
  $config: (
    // A map of CSS variable overrides to pass to the panel component if used
    // @type map
    "modal-panel-override": (
      "box-shadow": tokens.get("box-shadow-xl")
    ),

    // Map for outputting modal--size-[key] modifier variants
    // @type map
    "modal-size": (
      "default": 36em,
      "xs": 20em,
      "sm": 24em,
      "lg": 48em,
      "xl": 60em
    )
  )
);
```
</CodeExample>

## Basic usage

Modals are composed using classes and data attributes for their triggers. The basic structure of a modal is an element with a unique `id` and the `modal` class containing a child element with the `modal__dialog` class.

<CodeExample>
  <Fragment slot="output">
    <button class="link" command="show-modal" commandfor="modal-1">Open</button>
    <dialog id="modal-1" class="modal background border-radius padding" closedby="any">
      <div class="flex justify-content-between">
        <span>Modal example</span>
        <button class="link" command="close" commandfor="modal-1">Close</button>
      </div>
    </dialog>
  </Fragment>
  ```html
  <button command="show-modal" commandfor="[unique-id]">...</button>

  <dialog id="[unique-id]" class="modal" closedby="any">
    <button command="close" commandfor="[unique-id]">...</button>
    ...
  </dialog>
```
</CodeExample>

The dialog element of a modal is defined using the `modal__dialog` class. Modal dialogs are given the `role` attribute with a value of [`dialog`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/dialog_role) and the `aria-modal` attribute with a value of `true`. Authors should provide modal dialogs with [`aria-labelledby`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Attributes/aria-labelledby) and [`aria-describedby`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Attributes/aria-describedby) attributes if applicable to further improve accessibility.

<CodeExample>
  <Fragment slot="output">
    <button class="link" command="show-modal" commandfor="modal-2">Open</button>
    <dialog id="modal-2" class="modal panel" closedby="any" aria-labelledby="modal-2-title" aria-describedby="modal-2-description">
      <div class="panel__header">
        <h2 class="panel__title" id="modal-2-title">Modal panel</h2>
        <button class="link" command="close" commandfor="modal-2">Close</button>
      </div>
      <div class="panel__body">
        <p id="modal-2-description">Lorem ipsum dolor sit amet, consectetur adipiscing elit. Aenean porta diam eget lectus interdum, eu aliquet augue rutrum. Morbi faucibus mauris lectus, in imperdiet augue cursus vel.</p>
      </div>
      <div class="panel__footer">
        <button class="button button--color-primary" command="close" commandfor="modal-2">Confirm</button>
        <button class="button" command="close" commandfor="modal-2">Cancel</button>
      </div>
    </dialog>
  </Fragment>
  ```html
  <dialog id="[unique-id]" class="modal panel" closedby="any" aria-labelledby="dialog-title" aria-describedby="dialog-description">
    <div class="panel__header">
      <h2 class="panel__title" id="dialog-title">...</h2>
      <button class="link" command="close" commandfor="[unique-id]">Close</button>
    </div>
    <div class="panel__body">
      <p id="dialog-description">...</p>
    </div>
    <div class="panel__footer">
      ...
    </div>
  </dialog>
  ```
</CodeExample>

The [panel component](/packages/panel) is a great fit for styling a modal's dialog. This can be applied directly on the `modal` element and expanded as needed.

## modal--full

Modifier that adds styles to a modal to make it fill the entire viewport when opened.

<CodeExample>
  <Fragment slot="output">
    <button class="link" command="show-modal" commandfor="modal-6">Open</button>
    <dialog id="modal-6" class="modal modal--full panel" closedby="any" aria-labelledby="modal-6-title">
      <div class="panel__header">
        <h2 class="panel__title" id="modal-6-title">Full modal</h2>
        <button class="link" command="close" commandfor="modal-6">Close</button>
      </div>
      <div class="panel__body">
        <p>This modal should fill the entire viewport when opened.</p>
      </div>
      <div class="panel__footer">
        <button class="button button--color-primary" command="close" commandfor="modal-6">Accept</button>
        <button class="button" command="close" commandfor="modal-6">Cancel</button>
      </div>
    </dialog>
  </Fragment>
  ```html
  <dialog id="[unique-id]" class="modal modal--full" closedby="any">...</dialog>
  ```
</CodeExample>
