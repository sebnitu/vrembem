## Basic usage

The most basic implementation of the select component consists of the `.select` class applied to a `<select>` form-control element.

<CodeExample>
  <Fragment slot="output">
    <select class="select" name="example-1">
      <option value="">Select one</option>
      <option value="1">Option 1</option>
      <option value="2">Option 2</option>
      <option value="3">Option 3</option>
    </select>
  </Fragment>
  ```html
  <select class="select">
    <option value="">Select one</option>
    ...
  </select>
  ```
</CodeExample>

### Disabled

Adding the boolean `disabled` attribute to the select will provide visual indication that the form-control is not available for use.

<CodeExample>
  <Fragment slot="output">
    <select class="select" name="example-2">
      <option value="">Select one</option>
      <option value="1">Option 1</option>
      <option value="2">Option 2</option>
      <option value="3">Option 3</option>
    </select>
  </Fragment>
  ```html
  <select class="select" disabled>
    <option value="">Select one</option>
    ...
  </select>
  ```
</CodeExample>

### Invalid

Adding the `is-invalid` class or `aria-invalid="true"` attribute to the select will provide visual indication that the form-control value is invalid. Invalid state is also triggered by the native `:user-invalid` state.

<CodeExample>
  <Fragment slot="output">
    <select class="select is-invalid" name="example-2">
      <option value="">Select one</option>
      <option value="1">Option 1</option>
      <option value="2">Option 2</option>
      <option value="3">Option 3</option>
    </select>
  </Fragment>
  ```html
  <select class="select is-invalid">
    <option value="">Select one</option>
    ...
  </select>
  ```
</CodeExample>
