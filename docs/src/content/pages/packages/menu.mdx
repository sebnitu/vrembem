## Options

Menu is built as a Sass module. Sass options are used to control CSS output, while design token CSS variables control the styling of the component. Below are the available configuration options.

### Sass

Menu comes with the following Sass configuration options that can be set using either the [config module](/reference/sass-modules/config#using-config-module) or [Sass `with` clause](/reference/sass-modules/config#using-with-clause) methods.

<CodeExample lang="scss">
```scss
@use "@vrembem/menu" with (
  $config: (
    // Toggle for outputting menu--inline modifier breakpoint variants
    // @type boolean
    "menu-inline-breakpoints": true,

    // Toggle for outputting menu--full modifier breakpoint variants
    // @type boolean
    "menu-full-breakpoints": true
  )
);
```
</CodeExample>

## Basic usage

The menu component is composed of at minimum three parts: `menu`, `menu__item` and `menu__action`. The menu and menu items should be a `<ul>` and list items `<li>` respectively while the menu action can be either an `<a>` or `<button>` element. Also available is the optional `menu__sep` element to create separators in between menu items.

<CodeExample>
  <Fragment slot="output">
    <Menu name="edit" />
  </Fragment>
  ```html
  <ul class="menu">
    <li class="menu__item">
      <button class="menu__action">
        ...
      </button>
    </li>
    <li class="menu__sep"></li>
  </ul>
  ```
</CodeExample>

Use the `flex-grow` utility class to wrap text inside the `menu__action` element. Additional elements inside the menu action receive appropriate spacing.

<CodeExample>
  <Fragment slot="output">
    <Menu name="edit-all" />
  </Fragment>
  ```html
  <ul class="menu">
    <li class="menu__item">
      <button class="menu__action">
        <span>...</span>
        <span class="flex-grow">...</span>
        <span>...</span>
      </button>
    </li>
    ...
  </ul>
  ```
</CodeExample>

For links that only contain an icon, you can use the `menu__action_icon` element modifier to create a square link similar to the `button--icon` modifier.

<CodeExample>
  <Fragment slot="output">
    <Menu name="browser" />
  </Fragment>
  ```html
  <ul class="menu menu--inline">
    <li class="menu__item">
      <button class="menu__action menu__action_icon">
        ...
      </button>
    </li>
    ...
  </ul>
  ```
</CodeExample>

### is-active

Adding the `is-active` class will provide visual indication that the action is currently in an active state.

<CodeExample>
  <Fragment slot="output">
    <ul class="menu menu--inline">
      <li class="menu__item">
        <button class="menu__action is-active" disabled>One</button>
      </li>
      <li class="menu__item">
        <button class="menu__action">Two</button>
      </li>
      <li class="menu__item">
        <button class="menu__action">Three</button>
      </li>
    </ul>
  </Fragment>
  ```html
  <ul class="menu menu--inline">
    <li class="menu__item">
      <button class="menu__action is-active" disabled>
        ...
      </button>
    </li>
    ...
  </ul>
  ```
</CodeExample>

### disabled

Adding the boolean `disabled` attribute will provide visual indication that the user should not be able to interact with the action.

<CodeExample>
  <Fragment slot="output">
    <ul class="menu menu--inline">
      <li class="menu__item">
        <button class="menu__action" disabled>One</button>
      </li>
      <li class="menu__item">
        <button class="menu__action">Two</button>
      </li>
      <li class="menu__item">
        <button class="menu__action">Three</button>
      </li>
    </ul>
  </Fragment>
  ```html
  <ul class="menu menu--inline">
    <li class="menu__item">
      <button class="menu__action" disabled>
        ...
      </button>
    </li>
    ...
  </ul>
  ```
</CodeExample>

## menu--inline

Used to apply horizontal menu styles. This is typically used for short menus or toolbars where vertical space can be saved.

<CodeExample>
  <Fragment slot="output">
    <Menu name="crud" classes="menu--inline" />
  </Fragment>
  ```html
  <ul class="menu menu--inline">...</ul>
  ```
</CodeExample>

### Breakpoint variants

These modifiers come with media breakpoint variants. This allows changing styles based on a specific breakpoint key in the [`breakpoints`](/packages/core#config-options) config options map. Breakpoint variants output can be toggled using the `menu-inline-breakpoints` config option (default: `true`).

<CodeExample>
  <Fragment slot="demo">
    <label class="text-nowrap" for="demo-menu-inline-select">Variants Demo:</label>
    <select id="demo-menu-inline-select" class="select font-size-sm width-auto" data-demo-target="demo-menu-inline">
      <option value="xs:menu--inline">xs:menu--inline</option>
      <option value="sm:menu--inline">sm:menu--inline</option>
      <option value="md:menu--inline">md:menu--inline</option>
      <option value="lg:menu--inline">lg:menu--inline</option>
      <option value="xl:menu--inline">xl:menu--inline</option>
    </select>
  </Fragment>
  <div slot="output" class="spacing">
    <p>Current view-port width: <code data-vp-width>...</code></p>
    <Menu id="demo-menu-inline" name="crud" classes="lg:menu--inline" />
  </div>
  ```html
  <ul class="menu lg:menu--inline">...</ul>
  ```
</CodeExample>

**Available variants**

- `xs:menu--inline`
- `sm:menu--inline`
- `md:menu--inline`
- `lg:menu--inline`
- `xl:menu--inline`

## menu--full

Used to span a horizontal menu to fill the full width of its container. This modifier is meant to be paired with the `menu--inline` modifier as the default styles of a vertical menu already fill the full width of their container.

<CodeExample>
  <div slot="output" class="spacing">
    <Menu name="crud" classes="menu--inline menu--full" />
  </div>
  ```html
  <ul class="menu menu--inline menu--full">...</ul>
  ```
</CodeExample>

### Breakpoint variants

These modifiers come with media breakpoint variants. This allows changing styles based on a specific breakpoint key in the [`breakpoints`](/packages/core#config-options) config options map. Breakpoint variants output can be toggled using the `menu-full-breakpoints` config option (default: `true`).
