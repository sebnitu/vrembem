## Options

Core comes with the following Sass configuration options that can be set using either the [config module](/reference/sass-modules/config#using-config-module) or [Sass `with` clause](/reference/sass-modules/config#using-with-clause) methods.

<CodeExample lang="scss" expandable>
```scss
@use "@vrembem/core" with (
  $config: (
    // A prefix to apply to block class names. Provided value is automatically 
    // suffixed with a "-" character when applied.
    // @type string
    "prefix-blocks": "",

    // A prefix to apply to element class names
    // @type string
    "prefix-elements": "__",

    // A prefix to apply to modifier class names
    // @type string
    "prefix-modifiers": "_",

    // A prefix to apply to modifier value class names
    // @type string
    "prefix-modifier-values": "_",

    // A prefix to apply to CSS variable tokens. Map values requires a default
    // key and value along with any component specific prefixes as needed.
    // Provided value(s) are automatically suffixed with a "-" character.
    // @type string | map
    "prefix-tokens": "vb",

    // The palette to build CSS variables from
    // @type map
    "palette": (
      "primary": oklch(74% 40% 158deg),   // #35cc86
      "secondary": oklch(56% 16% 220deg), // #477f90
      "neutral": oklch(74% 4% 160deg),    // #9cb0a4
      "important": oklch(60% 40% 28deg),  // #ce5247
    ),

    // Number of decimal places to round colors converted to oklch color space.
    // @type number
    "palette-round-decimal": 4,

    // The key/value pairs of palette variants to create CSS variables for where 
    // key is used in the token variant name (--vb-primary-[key]) and value is 
    // passed to the palette get function as lightness/chroma arguments.
    // @type map
    "palette-variants": (
      "50":  (98%, 10%), // The lightest variant in the palette
      "75":  (96%, 20%),
      "100": (90%, 40%),
      "200": (80%, 80%),
      "300": 70%,
      "400": 60%,
      "500": 50%,
      "600": 40%,
      "700": 30%,
      "800": (20%, 80%),
      "900": (18%, 70%),
      "950": (16%, 60%), // The darkest variant in the palette
    ),

    // Sets the default theme option
    // @type string
    "theme-default": "light",

    // A map of breakpoint key/value pairs
    // @type map
    "breakpoints": (
      "xs": 480px,
      "sm": 620px,
      "md": 760px,
      "lg": 990px,
      "xl": 1380px
    )
  )
);
```
</CodeExample>

## Design tokens

Design tokens are the foundation of the Vrembem library and help to establish a consistent design language across components. Tokens are managed by the `@vrembem/core/tokens` module and are output as CSS variables.

[Design token reference](/reference/design-tokens)

## Sass modules

Vrembem uses Sass modules to manage global configuration and design tokens throughout the library.

- [`@vrembem/core/config`](/reference/sass-modules/config): A Sass module for managing global configuration options. This is available across all components to set things like BEM naming syntax, prefixes, and output options.
- [`@vrembem/core/tokens`](/reference/sass-modules/tokens): A Sass module to define, manage, and output global and component-level design tokens expressed as CSS variables.
  - `@vrembem/core/palette`: Adds token support for generating and accessing palette based tokens.
  - `@vrembem/core/breakpoints`: Adds token support for responsive values that change across breakpoints.
  - `@vrembem/core/themes`: Adds token support for themed values using light, dark, and custom themes.

## Collections

Collection and CollectionEntry are base classes designed to manage component instances—such as modals, drawers, or popovers. Each Collection acts as a controller, handling registration, events, lifecycle hooks, and plugin integration.

CollectionEntry represent an individual component instance within a collection, storing its state, configuration, and DOM reference. By extending these classes, Vrembem components inherit shared systems and APIs that can also be leveraged by custom components or plugins.

- [Collection Tutorial](/reference/collections/)
- [Collection API](/reference/collections/collection-api)
- [CollectionEntry API](/reference/collections/collection-entry-api)

### Plugins

Vrembem provides plugins designed to extend the features of components managed by collections, adding advanced behaviors and integrations to your JavaScript components.

- [`attrConfig`](/reference/collections/plugins/attr-config): Adds the ability to set a configuration source using the value of a data attribute.
- [`cssConfig`](/reference/collections/plugins/css-config): Adds the ability to set a configuration source using the values of CSS custom properties.
- [`focusTrap`](/reference/collections/plugins/focus-trap): Ensures keyboard focus remains within a specified element, such as a modal or dialog, while it is open. This improves accessibility by preventing users from tabbing outside interactive overlays.
- [`mediaQuery`](/reference/collections/plugins/media-query): Provides a flexible API for managing responsive behaviors and breakpoints using custom data attributes and media queries. It enables components to react to viewport changes and device characteristics.
- [`propStore`](/reference/collections/plugins/prop-store): Synchronizes component state with local storage, allowing values to persist across page reloads and sessions. It supports conditional storage and change callbacks for advanced state management.
- [`teleport`](/reference/collections/plugins/teleport): Moves DOM elements to different locations in the document, such as rendering modals or tooltips outside their parent containers. This helps manage stacking contexts and ensures overlays display correctly.

It's also possible to write your own custom plugins. Custom plugins allow you to extend collections with specialized logic, event handling, or integrations tailored to your specific requirements.

[Writing a custom plugin](/reference/collections/plugins/custom)
