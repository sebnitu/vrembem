# Base

A collection of modules that apply global styles and provide classes for styling commonly used HTML elements.

```sh
npm install @vrembem/base
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/base";
  ```
</CodeExample>

## Output

Base modules each have their own output styles. These are all output by default when the base package is loaded. There are two ways to control which modules are output using the `base-output` and `base-output-exceptions` configuration options.

<ReferenceCard ref="config" name="base-output" type="boolean" defaults="true">
  Controls whether module styles are output. You can specify exceptions using the `base-output-exceptions` list.

```scss
@include config.set("base-output", false);
```
</ReferenceCard>

<ReferenceCard ref="config" name="base-output-exceptions" type="list" defaults="()">
  Modules listed in this configuration are exceptions to the value set in `base-output`. If `base-output` is `true`, modules listed will not be output. If `base-output` is `false`, modules listed will be output.

```scss
@include config.set("base-output-exceptions", (
  "normalize", "headings"
));
```
</ReferenceCard>

#### Example

Use the `base-output` configuration to toggle output for all modules. Add exceptions to the `base-output-exceptions` list to override the output option for specific modules.

<CodeExample lang="scss">
```scss
// Import the config module
@use "@vrembem/core/config";

// Disable the output of specific modules
@include config.set("base-output-exceptions", (
  "blockquote",
  "code",
  "list",
  "pre"
));

// Disable the output of all modules except for "normalize" and "headings"
@include config.set((
  "base-output": false
  "base-output-exceptions": (
    "normalize",
    "headings"
  )
));
```
</CodeExample>

## Basic usage

Once loaded, the base component provides normalization and default styles for HTML elements along with a number of element classes. To style HTML elements, use their corresponding classes or apply the `type` class to style an entire block of content.

<CodeExample lang="html">
```html
<!-- Using base element modules -->
<h1 class="h1">...</h1>
<p>...</p>
<ul class="list">
  <li>...</li>
</ul>

<!-- Using base type module -->
<div class="type">
  <h1>...</h1>
  <p>...</p>
  <ul>
    <li>...</li>
  </ul>
</div>
```
</CodeExample>

You can also customize the class selectors used by modules by mapping the module name to your preferred class selector in the `base-classes` configuration.

<ReferenceCard ref="config" name="base-classes" type="map" defaults='("separator": "sep")'>
  A map containing module names as keys and their preferred class selectors as values. For example, to use the class `.content` instead of `.type`, you could set the following:

```scss
@include config.set("base-classes", (
  "separator": "sep",
  "type": "content"
));
```
</ReferenceCard>

## Modules

Below is a list of each base module, their class selectors, CSS variables, and intended usage. Modules that reference CSS variables are provided with sensible fallbacks.

### normalize

<ReferenceBar selector="n/a" hash={{
  target: "reference-table-cssvar-base",
  event: "filter",
  detail: {
    value: "--vb-base"
  }
}} />

A module that helps ensure HTML elements are rendered consistently across browsers by resetting and standardizing their default styles. This provides a reliable foundation for building components, making elements more predictable and easier to work with by:

- Resetting margin, padding, and box-sizing for all elements.
- Setting default html/body height, font, and background styles.
- Normalizing heading, image, table and caption elements.
- Creating consistent typography styles for code and pre elements.
- Normalizing form-control element appearance and font styles.

#### Variables

The following CSS variables are referenced with a fallback for more easily overriding base properties:

<ReferenceCard name="--vb-base-height" type="height value" fallback="100%">
  Used to set the height property on both the `html` and `body` elements.
</ReferenceCard>

<ReferenceCard name="--vb-base-background" type="color" fallback="var(--vb-background)">
  Used to set the background property on the `body` element.
</ReferenceCard>

<ReferenceCard name="--vb-base-foreground" type="color" fallback="var(--vb-foreground)">
  Used to set the color property on the `body` element.
</ReferenceCard>

<ReferenceCard name="--vb-base-font-size" type="font-size value" fallback="16px">
  Used to set the font-size property on the `html` element.
</ReferenceCard>

<ReferenceCard name="--vb-base-line-height" type="line-height value" fallback="var(--vb-line-height)">
  Used to set the line-height property on the `html` element.
</ReferenceCard>

Section headings in HTML are represented by the `<h1>` through `<h6>` elements. This module helps style these elements by providing the `.h1`-`.h6` CSS classes.

<CodeExample lang="html">
<Fragment slot="output">
  <div class="spacing">
    <h1 class="h1">Heading h1</h1>
    <h2 class="h2">Heading h2</h2>
    <h3 class="h3">Heading h3</h3>
    <h4 class="h4">Heading h4</h4>
    <h5 class="h5">Heading h5</h5>
    <h6 class="h6">Heading h6</h6>
  </div>
</Fragment>
```html
<h1 class="h1">...</h1>
<h2 class="h2">...</h2>
<h3 class="h3">...</h3>
<h4 class="h4">...</h4>
<h5 class="h5">...</h5>
<h6 class="h6">...</h6>
```
</CodeExample>

Headings output and scale is controlled by the `base-headings` configuration option.

<ReferenceCard ref="config" name="base-headings" type="map">
  Used to output the heading styles and their font-size/line-height CSS variables. This map should contain a key of the heading level to output and the value of a font-size and optional line-height (space separated).

```scss
@include config.set("base-headings", (
  "h1": 3em 1.2,
  "h2": 2em 1.6,
  // ...
));
```

  <hr class="sep" />
  <p class="font-family-mono font-size-sm">@default</p>

```scss
"base-headings": (
  "h1": 2.75em,
  "h2": 2em,
  "h3": 1.75em,
  "h4": 1.5em,
  "h5": 1.25em,
  "h6": 1em
)
```
</ReferenceCard>

### type

<ReferenceBar selector=".type" hash={{
  target: "reference-table-cssvar-base",
  event: "filter",
  detail: {
    value: "--vb-type"
  }
}} />

The type module provides the `.type` CSS class as a way to apply many base module styles to HTML elements directly. This is most commonly used to style a block of HTML content like an article or blog post.

<CodeExample lang="html">
<div slot="output" class="type spacing">
  <h1>Heading</h1>
  <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Nam ipsum ex, elementum ac dignissim quis, tempus non eros. Sed risus neque, tempus eget orci vel, sagittis lobortis ipsum.</p>
  <ul>
    <li>One</li>
    <li>Two</li>
    <li>Three</li>
  </ul>
</div>
```html
<div class="type">
  <h1>Heading</h1>
  <p>...</p>
  <ul>
    <li>...</li>
    <li>...</li>
    <li>...</li>
  </ul>
</div>
```
</CodeExample>

The type module only applies styles to its direct children using the `>` child combinator. This is to allow nesting of other components within a type block and avoid style conflicts. Module specific type selectors are controlled using the `base-type-selectors` variable map.

<ReferenceCard ref="config" name="base-type-selectors" type="map">
  Map modules to type selectors. This configuration is used when a module runs the `base.selector()` function. The keys of the map should represent a base module name and it's value the selector to be used within the `.type` context.

```scss
@include config.set("base-type-selectors", (
  "h1": "> h1",
  "h2": "> h2",
  "h3": "> h3",
  // ...
));
```

  <hr class="sep" />
  <p class="font-family-mono font-size-sm">@default</p>

```scss
"base-type-selectors": (
  "h1": "> h1",
  "h2": "> h2",
  "h3": "> h3",
  "h4": "> h4",
  "h5": "> h5",
  "h6": "> h6",
  "link": ":not(div) > a",
  "list": "> ul, > ol",
  "code": ":not(pre) > code",
  "pre": "> pre",
  "blockquote": "> blockquote",
  "separator": "> hr"
)
```
</ReferenceCard>

### arrow

<CodeExample>

<CodeExample lang="html">

### sr-only

<ReferenceBar selector=".sr-only" />

This module provides an accessibility class for rendering content meant only for screen readers. This keeps the content accessible in the DOM for screen readers while hiding it visually from all other users.

<CodeExample lang="html">
<Fragment slot="output">
<p>The following content is for screen readers only: <span class="sr-only">Hello, screen readers!</span></p>
</Fragment>
```html
<p>
  The following content is for screen readers only:
  <span class="sr-only">Hello, screen readers!</span>
</p>
```
</CodeExample>
