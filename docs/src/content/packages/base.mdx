# Base

A collection of modules that apply global style to commonly used HTML elements.

```sh
npm install @vrembem/base
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/base";
  ```
</CodeExample>

## Output

Base modules each have their own output styles. These are all output by default when the base package is loaded. There are two ways to control which modules are output using the `$output` variable and `$output-exceptions` list.

<ReferenceCard name="$output" type="boolean" defaults="true">
  Controls whether module styles are output. You can specify exceptions using the `$output-exceptions` list.
</ReferenceCard>

<ReferenceCard name="$output-exceptions" type="list" defaults="()">
  Modules listed in this variable are exceptions to the value set in `$output`. If `$output` is `true`, modules listed here will not be output. If `$output` is `false`, modules listed here will be output.
</ReferenceCard>

#### Example

Use the `$output` variable to toggle output for all modules. Add exceptions to the `$output-exceptions` list to override this setting for specific modules.

<CodeExample lang="scss">
```scss
// Disable the output of specific modules
@use "@vrembem/base" with (
  $output-exceptions: (
    "blockquote",
    "code",
    "list",
    "pre"
  )
);

// Disable the output of all modules except for "normalize"
@use "@vrembem/base" with (
  $output: false,
  $output-exceptions: (
    "normalize"
  )
);
```
</CodeExample>

## Basic usage

Once loaded, the base component provides default styles for HTML elements and a number of output modules. To style HTML elements, use their corresponding classes or apply the `type` class to style an entire block of content.

<CodeExample lang="html">
```html
<!-- Using base element modules -->
<h1 class="h1">...</h1>
<p>...</p>
<ul class="list">
  <li>...</li>
</ul>

<!-- Using base type module -->
<div class="type">
  <h1>...</h1>
  <p>...</p>
  <ul>
    <li>...</li>
  </ul>
</div>
```
</CodeExample>

You can also customize the class selectors used by modules by mapping the module name to your preferred class selector in the `$classes` map.

<ReferenceCard name="$classes" type="map" defaults='("separator": "sep")'>
  A map containing module names as keys and their preferred class selectors as values. For example, to use the class `.content` instead of `.type`, you could set the following:

```scss
@use "@vrembem/base" with (
  $classes: (
    "separator": "sep",
    "type": "content"
  )
);
```
</ReferenceCard>

## Modules

Below is a list of each base module, a description of what they do, their default class selectors (if any), and how to use them in your project.

### normalize

A module that helps ensure HTML elements are rendered consistently across browsers by resetting and standardizing their default styles. This provides a reliable foundation for building components, making elements more predictable and easier to work with by:

- Resetting margin, padding, and box-sizing for all elements.
- Setting default html/body height, font, and background styles.
- Normalizing heading, image, table and caption elements.
- Creating consistent typography styles for code and pre elements.
- Normalizing form-control element appearance and font styles.

#### Variables

The following CSS variables are referenced with a provided fallback for more easily overriding base properties:

<ReferenceCard name="--vb-base-height" type="height value" fallback="100%">
  Used to set the height property of both the `html` and `body` elements. The fallback value is set using the `$base-height` Sass variable.
</ReferenceCard>

<ReferenceCard name="--vb-base-background" type="color" fallback="var(--vb-background)">
  Used to set the background property of the `body` element. The fallback value is set using the `$base-background` Sass variable.
</ReferenceCard>

<ReferenceCard name="--vb-base-foreground" type="color" fallback="var(--vb-foreground)">
  Used to set the color property of the `body` element. The fallback value is set usign the `$base-foreground` Sass variable.
</ReferenceCard>

### blockquote

The HTML blockquote element is used for marking up extended quotations. This module helps style these elements in a distinct and appealing way by providing the `.blockquote` CSS class.

```html
<blockquote class="blockquote" cite="...">
  <p>...</p>
  <footer>
    ...
    <cite>...</cite>
  </footer>
</blockquote>
```

### code

The HTML code element displays its contents styled in a fashion intended to indicate that the text is a short fragment of computer code. This module helps style these elements by providing the `.code` CSS class.

```html
<code class="code">a = 17</code>
```

### heading

Section headings in HTML are represented by the `<h1>` through `<h6>` elements. This module helps style these elements by providing the `.h1`-`.h6` CSS classes.

```html
<h1 class="h1">...</h1>
<h2 class="h2">...</h2>
<h3 class="h3">...</h3>
<h4 class="h4">...</h4>
<h5 class="h5">...</h5>
<h6 class="h6">...</h6>
```

A map containing the font-size and optional line-height scale for HTML headings. The map should contain a key of the heading level and the value of a font-size and optional line-height space separated.

```scss
$heading-scale: (
  "h1": 2.25em,
  "h2": 2em,
  "h3": 1.75em,
  "h4": 1.5em,
  "h5": 1.25em inherit,
  "h6": 1em inherit
) !default;
```

### link

A link—usually represented by an anchor (`<a>`) HTML element with `href` attribute—creates the styles for a hyperlink to anything a URL can address. This module helps style these elements by providing the `.link` CSS class as well as a few optional modifiers.

```html
<a href="#" class="link">Link</a>
<a href="#" class="link link_subtle">Link</a>
<a href="#" class="link link_invert">Link</a>
<a href="#" class="link link_invert-subtle">Link</a>
```

### list

The list module helps add styles to unordered (`<ul>`) and ordered (`<ol>`) lists by providing the `.list` class.

```html
<ul class="list">
  <li>One</li>
  <li>Two
    <ul>
      <li>One</li>
      <li>Two</li>
      <li>Three</li>
    </ul>
  </li>
  <li>Three</li>
</ul>

<ol class="list">
  ...
</ol>
```

### pre

This module helps style the HTML `<pre>` element by providing the `.pre` CSS class. Whitespace inside this element is displayed as written.

```html
<pre class="pre">
-----------------------------
| I'm an expert in my field |
-----------------------------
     \   ^__^
      \  (oo)\_______
         (__)\       )\/\
             ||----w |
             ||     ||
</pre>
```

### separator

This module adds the `.sep` and `.sep-invert` CSS classes which visually renders a horizontal separator. This can be applied to an `<hr>` HTML element. It can also be applied to a more generic `<span>` or `<div>` depending on the semantic context.

```html
<!-- Separator -->
<hr class="sep">
<hr class="sep border-color-darker">

<!-- Invert separator -->
<hr class="sep-invert">
<hr class="sep-invert border-color-invert-darker">
```

### type

The type module provides the `.type` CSS class as a quick way to apply many base modules to HTML elements directly. Base module classes will override a parent `.type` application when explicitly set. Use `.type_invert` for when text sits on a dark background.

```html
<div class="type">
  <h1>Heading</h1>
  <p>...</p>
  <ul>
    <li>...</li>
    <li>...</li>
    <li>...</li>
  </ul>
</div>
```

Modules that get mapped to HTML elements include:

- `.h1`-`.h6` &rarr; `<h1>`-`<h6>`
- `.link` &rarr; `<a>`
- `.code` &rarr; `<code>`
- `.sep` &rarr; `<hr>`
- `.list` &rarr; `<ul>` `<ol>`
- `.blockquote` &rarr; `<blockquote>`
- `.pre` &rarr; `<pre>`

> The type module only applies styles to children HTML elements. It's possible to nest other components within type and not have style or specificity conflicts. It's recommended to wrap nested components with anonymous `<div>` elements when possible.

### arrow

The arrow (caret) module creates directional triangles drawn with CSS.

<CodeExample>
  <div slot="output" class="flex">
    <span class="arrow"></span>
    <span class="arrow-up"></span>
    <span class="arrow-left"></span>
    <span class="arrow-right"></span>
  </div>
  ```html
  <span class="arrow"></span>
  <span class="arrow-up"></span>
  <span class="arrow-left"></span>
  <span class="arrow-right"></span>
  ```
</CodeExample>

#### Example

Arrows are great indicators for buttons and menu items when interacting with them would toggle a popover or other togglable components.

```html
<button class="button">
  <span>Button</span>
  <span class="arrow"></span>
</button>
```
