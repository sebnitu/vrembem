# Core

Vrembem's core package provides a number of Sass and JavaScript modules that make up the shared systems, configurations, and utilities of the Vrembem library.

```sh
npm install @vrembem/core
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/core";
  ```
</CodeExample>

<CodeExample lang="js">
  ```js
  import * as core from "@vrembem/core";
  ```
</CodeExample>

## Config options

Core comes with the following configuration options that can be set using either the [config module](/packages/core/config#using-config-module) or [Sass with clause](/packages/core/config#using-with-clause) methods.

```scss
$config: (
  // A prefix to apply to block class names
  // @type string
  "prefix-block": "",

  // A prefix to apply to element class names
  // @type string
  "prefix-element": "__",

  // A prefix to apply to modifier class names
  // @type string
  "prefix-modifier": "_",

  // A prefix to apply to modifier value class names
  // @type string
  "prefix-modifier-value": "_",

  // A prefix to apply to utility class names
  // @type string
  "prefix-utility": "",

  // A prefix to apply to CSS variables
  // @type string
  "prefix-cssvar": "vb",

  // The palette to build CSS variables from
  // @type map
  "palette": (
    "primary": hsl(152deg 60% 50%),
    "secondary": hsl(214deg 50% 50%),
    "neutral": hsl(214deg 20% 50%),
    "important": hsl(0deg 80% 50%)
  ),

  // The default color lightness to use when referencing a color
  // @type number
  "palette-default": null,

  // A prefix to apply to theme names
  // @type string
  "theme-prefix": "vb-theme",

  // Sets the default theme option
  // @type string
  "theme-default": "light",
);
```

## Sass modules

Vrembem uses Sass modules to manage global configuration and custom properties throughout the library.

- [`@vrembem/core/config`](/packages/core/config): Used to setup global configuration options. This is available across all components to set things like BEM naming syntax, prefixes, and theme defaults.
- [`@vrembem/core/css`](/packages/core/css): Used to define, manage, and output global and component-level custom properties.
- [`@vrembem/core/palette`](/packages/core/palette): Used to build and return color palette values by leveraging both config and css modules.
- [`@vrembem/core/theme`](/packages/core/theme): Used to define, manage, and output global and component-level themes powered by custom properties.

### Theme state

After defining themes using Vrembem's core theme module, users should be able to select their preferred theme and have that choice persist. Vrembem provides `themeStore`, a JavaScript utility that manages theme state for your application or website.

[Getting started with themeStore](/packages/core/theme-store)

## Collections

### Plugins

Vrembem provides plugins designed to extend the features of components managed by collections, adding advanced behaviors and integrations to your JavaScript components.

- [`attrConfig`](/packages/core/attr-config): Adds the ability to set a configuration source using the value of a data attribute.
- [`cssConfig`](/packages/core/css-config): Adds the ability to set a configuration source using the values of CSS custom properties.
- [`focusTrap`](/packages/core/focus-trap): Ensures keyboard focus remains within a specified element, such as a modal or dialog, while it is open. This improves accessibility by preventing users from tabbing outside interactive overlays.
- [`mediaQuery`](/packages/core/media-query): Provides a flexible API for managing responsive behaviors and breakpoints using custom data attributes and media queries. It enables components to react to viewport changes and device characteristics.
- [`propStore`](/packages/core/prop-store): Synchronizes component state with local storage, allowing values to persist across page reloads and sessions. It supports conditional storage and change callbacks for advanced state management.
- [`teleport`](/packages/core/teleport): Moves DOM elements to different locations in the document, such as rendering modals or tooltips outside their parent containers. This helps manage stacking contexts and ensures overlays display correctly.

It's also possible to write your own custom plugins. Custom plugins allow you to extend collections with specialized logic, event handling, or integrations tailored to your specific requirements.

[Writing a custom plugin](/packages/core/custom-plugins)

## Reference

Outside of core systems, Vrembem also makes available a number of variables, functions and mixins that can be used throughout your project as needed.

### CSS Variables

Vrembem core defines a set of global variables via CSS custom properties that are accessible to all components and help establish a consistent design language across your project. These are stored under the `core` module and are not prefixed with the module name when output.

[CSS variables reference](/packages/core/css-variables)

### Sass Utilities

Vrembem core provides a number of helpful Sass functions and mixin utilities. Some of which are used internally by components to apply consistent design patterns (such as focus rings) and others are helpful stand-off utilities like drawing CSS arrows or loading spinners.

[Sass utilities reference](/packages/core/sass-utilities)
