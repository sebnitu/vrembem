# Media

The media component is used for displaying groups of content with a corresponding media asset, such as an image, avatar or icon.

```sh
npm install @vrembem/media
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/media";
  ```
</CodeExample>

## Basic usage

The most basic implementation of the media component consists of the `media` container and atleast one body element (`media__body`) and one object element (`media__obj`).

<CodeExample>
  <Fragment slot="output">
    <div class="media">
      <div class="media__obj">
        <img class="radius" src="https://picsum.photos/id/64/64/64" />
      </div>
      <div class="media__body">
        <h2 class="font-size-lg">Media example</h2>
        <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed libero nulla, suscipit consectetur lectus eu, rhoncus eleifend erat.</p>
      </div>
    </div>
  </Fragment>
  ```html
  <div class="media">
    <div class="media__obj">
      ...
    </div>
    <div class="media__body">
      ...
    </div>
  </div>
  ```
</CodeExample>

> Media is a flex based layout component. Use flex utilities from [`@vremben/utility`](/packages/utility) to further customize your layout. 

## media_gap_[key]

Adjusts the gap spacing based on the provided key. Gap key output is based on the values in [`$gap-map`](#scss-variables-file) variable map.

<CodeExample>
  <Fragment slot="output">
    <div class="media media_gap_sm">
      <div class="media__obj">
        <Icon name="settings" classes="foreground-primary" />
      </div>
      <div class="media__body">
        <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed libero nulla, suscipit consectetur lectus eu, rhoncus eleifend erat.</p>
      </div>
    </div>
  </Fragment>
  ```html
  <div class="media media_gap_sm">
    ...
  </div>
  ```
</CodeExample>

**Available variants**

- `media_gap_none`
- `media_gap_xs`
- `media_gap_sm`
- `media_gap_md`
- `media_gap_lg`
- `media_gap_xl`

## media_gap-y_[key]

Adjusts the vertical gap spacing based on the provided key. Gap key output is based on the values in [`$gap-map`](#scss-variables-file) variable map.

<CodeExample>
  <Fragment slot="output">
    <div class="media media_gap-y_xs media_stack">
      <div class="media__obj">
        <img class="radius" src="https://picsum.photos/id/64/64/64" />
      </div>
      <div class="media__body">
        <h2 class="font-size-lg">Media example</h2>
        <p>Lorem ipsum dolor sit amet, consectetur adipiscing elit. Sed libero nulla, suscipit consectetur lectus eu, rhoncus eleifend erat.</p>
      </div>
    </div>
  </Fragment>
  ```html
  <div class="media media_gap-y_xs media_stack">
    ...
  </div>
  ```
</CodeExample>

> The `media_gap-y_[key]` modifier only take effect when combined with a [`media_stack_[key]`](#media_stack_key) modifier.

**Available variants**

- `media_gap-y_none`
- `media_gap-y_xs`
- `media_gap-y_sm`
- `media_gap-y_md`
- `media_gap-y_lg`
- `media_gap-y_xl`

## media_stack_[key]

Adds a breakpoint for when media items should be stacked vertically. Values and class keys are generated using the [`$breakpoints`](#scss-variables-file) map. Omitting the key value from the modifier (e.g. `media_stack`) will stack items under all conditions.

<CodeExample>
  <div slot="output" class="gap-y">
    <p>Current view-port width: <code data-vp-width>...</code></p>
    <div class="media media_stack_lg">
      <div class="media__obj">
        <img class="radius" src="https://picsum.photos/id/64/64/64" />
      </div>
      <div class="media__body">
        <h2 class="font-size-lg">Media stack example</h2>
        <p>Media will stack below the <code>lg (990px)</code> breakpoint.</p>
      </div>
    </div>
  </div>
  ```html
  <div class="media media_stack_lg">
    ...
  </div>
  ```
</CodeExample>

**Available variants**

- `media_stack`
- `media_stack_xs`
- `media_stack_sm`
- `media_stack_md`
- `media_stack_lg`
- `media_stack_xl`
