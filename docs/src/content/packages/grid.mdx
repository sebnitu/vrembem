# Grid

A simple grid utility component.

```sh
npm install @vrembem/grid
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/grid";
  ```
</CodeExample>

## Basic usage

The most basic implementation of the grid layout consists of the grid container and it's children (grid items). Apply the `grid` utility class to the desired grid container with an optional `grid-cols-[value]` modifier to create a grid layout.

<CodeExample>
  <Fragment slot="output">
    <div class="grid grid-cols-3">
      <div class="box">1</div>
      <div class="box">2</div>
      <div class="box">3</div>
    </div>
  </Fragment>
  ```html
  <div class="grid grid-cols-3">
    <div>...</div>
    <div>...</div>
    <div>...</div>
  </div>
  ```
</CodeExample>

### grid-inline

Also available is the `grid-inline` utility. This applies the same styles as `grid` but will set the `display` property to `inline-grid` instead of `grid`.

<CodeExample>
  <Fragment slot="output">
    <div class="grid-inline grid-cols-3">
      <div class="box">1</div>
      <div class="box">2</div>
      <div class="box">3</div>
    </div>
  </Fragment>
  ```html
  <div class="grid-inline grid-cols-3">
    <div>...</div>
    <div>...</div>
    <div>...</div>
  </div>
  ```
</CodeExample>

## grid-cols-[value]

By default the `grid` utility class only renders a single column. To add additional columns to the grid layout, use the `grid-cols-[value]` modifier. The number of variants available is controlled by the `grid.$cols` variable (defaults to `6`).

<CodeExample>
  <Fragment slot="demo">
    <label for="demo-grid-cols-select" class="text-nowrap">Variants Demo:</label>
    <select id="demo-grid-cols-select" class="input input_type_select input_size_sm input_auto" data-demo-target="demo-grid-cols">
      <option value="grid">None</option>
      <option value="grid grid-cols-1">grid-cols-1</option>
      <option value="grid grid-cols-2">grid-cols-2</option>
      <option value="grid grid-cols-3">grid-cols-3</option>
      <option value="grid grid-cols-4">grid-cols-4</option>
      <option value="grid grid-cols-5">grid-cols-5</option>
      <option value="grid grid-cols-6">grid-cols-6</option>
    </select>
  </Fragment>
  <Fragment slot="output">
    <div id="demo-grid-cols" class="grid grid-cols-3">
      <div class="box">1</div>
      <div class="box">2</div>
      <div class="box">3</div>
      <div class="box">4</div>
      <div class="box">5</div>
      <div class="box">6</div>
    </div>
  </Fragment>
  ```html
  <div class="grid grid-cols-3">
    ...
  </div>
  ```
</CodeExample>

**Available variants**

- `grid-cols-1`
- `grid-cols-2`
- `grid-cols-3`
- `grid-cols-4`
- `grid-cols-5`
- `grid-cols-6`

## Customization

Grid is primarily customized using CSS variables but can also be modified using Sass variables. All CSS variables are given the Vrembem vendor prefix `"vb-"`. This can be changed by setting the `$prefix-variable` variable via the `core` module.

<DocBlock name="CSS Variables" src="grid/src/_root.scss">

```scss
:root {
  --vb-grid-cols: 1;
  --vb-grid-rows: 1;
  --vb-grid-flow: dense;
  --vb-grid-gap: 2em;
}
```
</DocBlock>

<DocBlock name="SCSS Variables" src="grid/src/_variables.scss">

```scss
@use "@vrembem/core";
@use "@vrembem/core/css";

$breakpoints: core.$breakpoints !default;
$cols: 6 !default;
$rows: 6 !default;
$flow: dense !default;
$gap: 2em !default;
$gap-map: (
  "none": 0,
  "xs": 0.5em,
  "sm": 1em,
  "md": 2em,
  "lg": 3em,
  "xl": 4em
) !default;

@include css.set("grid", (
  "cols": 1,
  "rows": 1,
  "flow": $flow,
  "gap": $gap,
  "gap-x": null,
  "gap-y": null,
));

```
</DocBlock>
