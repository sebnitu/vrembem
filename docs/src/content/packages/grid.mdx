# Grid

A simple grid utility component.

```sh
npm install @vrembem/grid
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/grid";
  ```
</CodeExample>

## Basic usage

The most basic implementation of the grid layout consists of the grid container and it's children (grid items). Apply the `grid` utility class to the desired grid container with an optional `grid-columns-[value]` modifier to create a grid layout.

<CodeExample>
  <Fragment slot="output">
    <div class="grid grid-columns-3">
      <div class="box">1</div>
      <div class="box">2</div>
      <div class="box">3</div>
    </div>
  </Fragment>
  ```html
  <div class="grid grid-columns-3">
    <div>...</div>
    <div>...</div>
    <div>...</div>
  </div>
  ```
</CodeExample>

### grid-inline

Also available is the `grid-inline` utility. This applies the same styles as `grid` but will set the `display` property to `inline-grid` instead of `grid`.

<CodeExample>
  <Fragment slot="output">
    <div class="grid-inline grid-columns-3">
      <div class="box">1</div>
      <div class="box">2</div>
      <div class="box">3</div>
    </div>
  </Fragment>
  ```html
  <div class="grid-inline grid-columns-3">
    <div>...</div>
    <div>...</div>
    <div>...</div>
  </div>
  ```
</CodeExample>

## grid-columns-[value]

By default the `grid` utility class only renders a single column. To add additional columns to the grid layout, use the `grid-columns-[value]` modifier. The number of variants available is controlled by the `grid.$columns` variable.

<CodeExample>
  <Fragment slot="demo">

    <select id="demo-grid-columns-select" class="input input_type_select input_size_sm input_auto" data-demo-target="demo-grid-columns">
      <option value="grid">None</option>
      <option value="grid grid-columns-2">grid-columns-2</option>
      <option value="grid grid-columns-3">grid-columns-3</option>
      <option value="grid grid-columns-4">grid-columns-4</option>
      <option value="grid grid-columns-5">grid-columns-5</option>
      <option value="grid grid-columns-6">grid-columns-6</option>
    </select>
  </Fragment>
  <Fragment slot="output">
    <div id="demo-grid-columns" class="grid grid-columns-3">
      <div class="box">1</div>
      <div class="box">2</div>
      <div class="box">3</div>
      <div class="box">4</div>
      <div class="box">5</div>
      <div class="box">6</div>
    </div>
  </Fragment>
  ```html
  <div class="grid grid-columns-3">
    ...
  </div>
  ```
</CodeExample>

**Available variants**

- `grid-columns-2`
- `grid-columns-3`
- `grid-columns-4`
- `grid-columns-5`
- `grid-columns-6`

## Customization

Grid is primarily customized using CSS variables but can also be modified using Sass variables. All CSS variables are given the Vrembem vendor prefix `"vb-"`. This can be changed by setting the `$prefix-variable` variable via the `core` module.

<DocBlock name="CSS Variables" src="grid/src/_root.scss">

```scss
:root {
  --vb-grid-columns: 1;
  --vb-grid-flow: dense;
  --vb-grid-gap: 2em;
}
```
</DocBlock>

<DocBlock name="SCSS Variables" src="grid/src/_variables.scss">

```scss
@use "@vrembem/core";
@use "@vrembem/core/css";

$breakpoints: core.$breakpoints !default;
$columns: 6 !default;
$flow: dense !default;
$gap: 2em !default;
$gap-map: (
  "none": 0,
  "xs": 0.5em,
  "sm": 1em,
  "md": 2em,
  "lg": 3em,
  "xl": 4em
) !default;

@include css.set("grid", (
  "columns": 1,
  "flow": $flow,
  "gap": $gap,
  "gap-x": null,
  "gap-y": null,
));
```
</DocBlock>
