# Grid

A simple grid based layout component.

```sh
npm install @vrembem/grid
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/grid";
  ```
</CodeExample>

## Basic usage

The most basic implementation of the grid layout consists of the grid container and it's children (grid items). Apply the `grid` utility class to the desired grid container with an optional `grid_cols_[value]` modifier to create a grid layout.

<CodeExample>
  <Fragment slot="output">
    <div class="grid grid_cols_3">
      <div class="box">1</div>
      <div class="box">2</div>
      <div class="box">3</div>
    </div>
  </Fragment>
  ```html
  <div class="grid grid_cols_3">
    <div>...</div>
    <div>...</div>
    <div>...</div>
  </div>
  ```
</CodeExample>

### grid_inline

Also available is the `grid_inline` modifier. This sets the `display` property to `inline-grid` instead of `grid`.

<CodeExample>
  <Fragment slot="output">
    <div class="grid grid_inline grid_cols_3">
      <div class="box">1</div>
      <div class="box">2</div>
      <div class="box">3</div>
    </div>
  </Fragment>
  ```html
  <div class="grid grid_inline grid_cols_3">
    <div>...</div>
    <div>...</div>
    <div>...</div>
  </div>
  ```
</CodeExample>

## grid_cols_[value]

By default the `grid` utility class only renders a single column. To add additional columns to the grid layout, use the `grid_cols_[value]` modifier. The number of variants available is controlled by the `grid.$cols` variable (defaults to `6`).

<CodeExample>
  <Fragment slot="demo">
    <label for="demo-grid-cols-select" class="text-nowrap">Variants Demo:</label>
    <select id="demo-grid-cols-select" class="input input_type_select input_size_sm input_auto" data-demo-target="demo-grid-cols">
      <option value="grid">None</option>
      <option value="grid grid_cols_1">grid_cols_1</option>
      <option value="grid grid_cols_2">grid_cols_2</option>
      <option value="grid grid_cols_3">grid_cols_3</option>
      <option value="grid grid_cols_4">grid_cols_4</option>
      <option value="grid grid_cols_5">grid_cols_5</option>
      <option value="grid grid_cols_6">grid_cols_6</option>
    </select>
  </Fragment>
  <Fragment slot="output">
    <div id="demo-grid-cols" class="grid grid_cols_3">
      <div class="box">1</div>
      <div class="box">2</div>
      <div class="box">3</div>
      <div class="box">4</div>
      <div class="box">5</div>
      <div class="box">6</div>
    </div>
  </Fragment>
  ```html
  <div class="grid grid_cols_3">
    ...
  </div>
  ```
</CodeExample>

**Available variants**

- `grid_cols_1`
- `grid_cols_2`
- `grid_cols_3`
- `grid_cols_4`
- `grid_cols_5`
- `grid_cols_6`

## grid_flow_[value]

Used to control the grid-auto-flow property which sets how auto-placed items get flowed into the grid. The default grid-auto-flow value is set to `dense` by the `grid.$flow` variable applied via the `--vb-grid-flow` custom property.

## Customization

Grid is primarily customized using CSS variables but can also be modified using Sass variables. All CSS variables are given the Vrembem vendor prefix `"vb-"`. This can be changed by setting the `$prefix-variable` variable via the `core` module.

<DocBlock name="CSS Variables" src="grid/src/_root.scss">

```scss
:root {
  --vb-grid-cols: 1;
  --vb-grid-rows: 1;
  --vb-grid-flow: dense;
  --vb-grid-gap: 2em;
}
```
</DocBlock>

<DocBlock name="SCSS Variables" src="grid/src/_variables.scss">

```scss
@use "@vrembem/core";
@use "@vrembem/core/css";

$breakpoints: core.$breakpoints !default;
$cols: 6 !default;
$rows: 6 !default;
$flow: dense !default;
$gap: 2em !default;
$gap-map: (
  "none": 0,
  "xs": 0.5em,
  "sm": 1em,
  "md": 2em,
  "lg": 3em,
  "xl": 4em
) !default;

@include css.set("grid", (
  "cols": 1,
  "rows": 1,
  "flow": $flow,
  "gap": $gap,
  "gap-x": null,
  "gap-y": null,
));
```
</DocBlock>
