# Button

Buttons are a simple component that allow users to take actions.

```sh
npm install @vrembem/button
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/button";
  ```
</CodeExample>

## Config options

Button comes with the following configuration options that can be set using either the [config module](/packages/core/config#using-config-module) or [Sass with clause](/packages/core/config#using-with-clause) methods.

<CodeExpander>
```scss
$config: (
  // Toggle for outputting button_block modifier breakpoint variants
  // @type boolean
  "button-block-breakpoints": true,

  // Map for outputting button_color_[key] modifier variants
  // @type map
  "button-color": (
    "primary": palette.get("primary"),
    "secondary": palette.get("secondary"),
    "neutral": palette.get("neutral"),
    "important": palette.get("important")
  ),

  // Map for outputting button_size_[key] modifier variants
  // @type map
  "button-size": (
    "breakpoints": true,
    "sm": css.get("form-control-font-size-sm"),
    "lg": css.get("form-control-font-size-lg")
  )
);
```
</CodeExpander>

## Basic usage

The most basic implementation of the button component consists of the `button` class that can be applied to `<button>`, `<a role="button">` or `<input>` elements with a type attribute of "button", "submit" or "reset". [Remember to add `role="button"`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/button_role) to any none semantic buttons as needed.

<CodeExample>
  <div slot="output" class="flex">
    <button class="button">Button</button>
    <a class="button" role="button" href="#">Anchor</a>
    <input class="button" type="button" value="Input" />
    <input class="button" type="submit" value="Submit" />
    <input class="button" type="reset" value="Reset" />
  </div>
  ```html
  <button class="button">Button</button>
  <a class="button" role="button">Anchor</a>
  <input class="button" type="button" value="Input" />
  <input class="button" type="submit" value="Submit" />
  <input class="button" type="reset" value="Reset" />
  ```
</CodeExample>

Elements within a button are centered vertically and spaced accordingly using the value set in `$gap` (defaults to `0.5em`). When elements are added inside a button, the button's text should also be wrapped in a `<span>` so that it can be spaced properly.

<CodeExample>
  <div slot="output">
    <button class="button">
      <Icon name="anchor" classes="icon_size_sm" />
      <span>Button</span>
      <span class="arrow"></span>
    </button>
  </div>
  ```html
  <button class="button">
    <svg class="icon icon_size_sm">...</svg>
    <span>Button</span>
    <span class="arrow"></span>
  </button>
  ```
</CodeExample>

### Disabled

When disabled using the `disabled` attribute, a button will inherit styles to visually appear noninteractive.

<CodeExample>
  <div slot="output">
    <button class="button" disabled>Button</button>
  </div>
  ```html
  <button class="button" disabled>Button</button>
  ```
</CodeExample>

### Loading

Buttons can also be given a loading state by adding the `is-loading` state class. This is useful when a button has an action that has a delayed response in order to give users a visual indicator that their action is being processed.

<CodeExample>
  <div slot="output">
    <button class="button is-loading" disabled>Button</button>
  </div>
  ```html
  <button class="button is-loading" disabled>Button</button>
  ```
</CodeExample>

> Warning: the `is-loading` state class will not work on `<input type="button">` buttons since it uses the `::after` pseudo-element to display the loading spinner.

## button_block

Gives a button "block" styles so that it spans the full width of its container. Values and class keys are generated using the [`$breakpoints`](#scss-variables-file) map. Using `button_block` will apply block styles in all viewports.

<CodeExample>
  <div slot="output">
    <button class="button button_block">Button</button>
  </div>
  ```html
  <button class="button button_block">Button</button>
  ```
</CodeExample>

**Available variants**

- `button_block`
- `xs:button_block`
- `sm:button_block`
- `md:button_block`
- `lg:button_block`
- `xl:button_block`

## button_color_[key]

Adds styles for changing the look and feel of a button. These are usually done with different background and text colors.

<CodeExample>
  <div slot="output" class="flex">
    <button class="button button_color_primary">Button</button>
    <button class="button button_color_secondary">Button</button>
    <button class="button button_color_neutral">Button</button>
    <button class="button button_color_important">Button</button>
  </div>
  ```html
  <button class="button button_color_primary">Button</button>
  <button class="button button_color_secondary">Button</button>
  <button class="button button_color_neutral">Button</button>
  <button class="button button_color_important">Button</button>
  ```
</CodeExample>

**Available variants**

- `button_color_primary`
- `button_color_secondary`
- `button_color_neutral`
- `button_color_important`

## button_icon

Adds styles that make icon-only buttons have equal padding. Will be square if the icon used also has equal width and height.

<CodeExample>
  <div slot="output">
    <button class="button button_icon">
      <Icon name="anchor" />
    </button>
  </div>
  ```html
  <button class="button button_icon">
    <svg class="icon">...</svg>
  </button>
  ```
</CodeExample>

## button_size_[key]

Adjust the size of a button by increasing or decreasing its padding and font-size.

<CodeExample>
  <div slot="output" class="flex align-items-center">
    <button class="button button_size_sm">Button</button>
    <button class="button">Button</button>
    <button class="button button_size_lg">Button</button>
  </div>
  ```html
  <button class="button button_size_sm">Button</button>
  <button class="button button_size_lg">Button</button>
  ```
</CodeExample>

**Available variants**

- `button_size_sm`
- `button_size_lg`
