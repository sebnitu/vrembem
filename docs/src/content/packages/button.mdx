# Button

Buttons are a simple component that allow users to take actions.

```sh
npm install @vrembem/button
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/button";
  ```
</CodeExample>

## Basic usage

The most basic implementation of the button component consists of the `button` class that can be applied to `<button>`, `<a role="button">` or `<input>` elements with a type attribute of "button", "submit" or "reset". [Remember to add `role="button"`](https://developer.mozilla.org/en-US/docs/Web/Accessibility/ARIA/Roles/button_role) to any none semantic buttons as needed.

<CodeExample>
  <div slot="output" class="level">
    <button class="button">Button</button>
    <a class="button" role="button">Anchor</a>
    <input class="button" type="button" value="Input" />
    <input class="button" type="submit" value="Submit" />
    <input class="button" type="reset" value="Reset" />
  </div>
  ```html
  <button class="button">Button</button>
  <a class="button" role="button">Anchor</a>
  <input class="button" type="button" value="Input" />
  <input class="button" type="submit" value="Submit" />
  <input class="button" type="reset" value="Reset" />
  ```
</CodeExample>

Elements within a button are centered vertically and spaced accordingly using the value set in `$gap` (defaults to `0.5em`). When elements are added inside a button, the button's text should also be wrapped in a `<span>` so that it can be spaced properly.

<CodeExample>
  <div slot="output">
    <button class="button">
      <Icon name="anchor" iconClass="icon_size_sm" />
      <span>Button</span>
      <span class="arrow"></span>
    </button>
  </div>
  ```html
  <button class="button">
    <svg class="icon icon_size_sm">...</svg>
    <span>Button</span>
    <span class="arrow"></span>
  </button>
  ```
</CodeExample>

### Disabled

When disabled using the `disabled` attribute, a button will inherit styles to visually appear noninteractive.

<CodeExample>
  <div slot="output">
    <button class="button" disabled>Button</button>
  </div>
  ```html
  <button class="button" disabled>Button</button>
  ```
</CodeExample>

### Loading

Buttons can also have a loading state by adding the `is-loading` state class. This is useful when a button has an action that has a delayed response in order to give users a visual indicator that their action is being processed.

<CodeExample>
  <div slot="output">
    <button class="button is-loading" disabled>Button</button>
  </div>
  ```html
  <button class="button is-loading" disabled>Button</button>
  ```
</CodeExample>

> Warning: the `is-loading` state class will not work on `<input type="button">` buttons since it uses the `::after` pseudo-element to display the loading spinner.

## button_block_[value]

Gives a button "block" styles so that it spans the full width of its container. Values and class keys are generated using the [`$breakpoints`](#breakpoints) map. Omitting the key value from the modifier (e.g. `button_block`) will apply block styles under all conditions.

<CodeExample>
  <div slot="output">
    <button class="button button_block">Button</button>
  </div>
  ```html
  <button class="button button_block">Button</button>
  ```
</CodeExample>

**Available Variants**

- `button_block`
- `button_block_xs`
- `button_block_sm`
- `button_block_md`
- `button_block_lg`
- `button_block_xl`

## button_color_[value]

Adds styles for changing the look and feel of a button. These are usually done with different background and text colors.

<CodeExample>
  <div slot="output" class="level">
    <button class="button button_color_primary">Button</button>
    <button class="button button_color_secondary">Button</button>
  </div>
  ```html
  <button class="button button_color_primary">Button</button>
  <button class="button button_color_secondary">Button</button>
  ```
</CodeExample>

**Available Variants**

- `button_color_primary`
- `button_color_secondary`

## button_icon

Adds styles that make icon-only buttons have equal padding. Will be square if the icon used also has equal width and height.

<CodeExample>
  <div slot="output">
    <button class="button button_icon">
      <Icon name="anchor" />
    </button>
  </div>
  ```html
  <button class="button button_icon">
    <svg class="icon">...</svg>
  </button>
  ```
</CodeExample>

## button_size_[value]

Adjust the size of a button by increasing or decreasing its padding and font-size.

<CodeExample>
  <div slot="output" class="level">
    <button class="button button_size_lg">Button</button>
    <button class="button">Button</button>
    <button class="button button_size_sm">Button</button>
  </div>
  ```html
  <button class="button button_size_lg">Button</button>
  <button class="button button_size_sm">Button</button>
  ```
</CodeExample>

**Available Variants**

- `button_size_sm`
- `button_size_lg`

## Customization

Buttons are primarily customized using CSS variables but can also be modified using Sass variables.

### CSS Variables

All CSS variables are given the Vrembem vendor prefix `"vb-"`. This can be changed by setting the `$prefix-variable` variable either globally via the `core` module or per component.

<div class="border-top">

  <DataCard 
    id="css-var-button-size"
    name="--vb-button-size"
    description="Used to set the min-width and height properties of buttons."
    value="button.$size"
    computed="2.5rem"
  />

  <DataCard 
    id="css-var-button-padding"
    name="--vb-button-padding"
    value="button.$padding"
    computed="calc(0.5em - 1px) 1em"
  />

  <DataCard 
    id="css-var-button-gap"
    name="--vb-button-gap"
    value="button.$gap"
    computed="0.5rem"
  />

  <DataCard
    id="css-var-border-width"
    name="--vb-button-border-width"
    value="button.$border-width"
    computed="1px"
  />

  <DataCard
    id="css-var-border-radius"
    name="--vb-button-border-radius"
    value="button.$border-radius"
    computed="1px"
  />

</div>

  --vb-button-size: #{var.$size};

  --vb-button-border-radius: #{var.$border-radius};
  --vb-button-font-size: #{var.$font-size};
  --vb-button-font-weight: #{var.$font-weight};
  --vb-button-line-height: #{var.$line-height};
  --vb-button-transition-property: #{var.$transition-property};
  --vb-button-transition-duration: #{var.$transition-duration};
  --vb-button-transition-timing-function: #{var.$transition-timing-function};
  --vb-button-background: #{var.$background};
  --vb-button-background-hover: #{var.$background-hover};
  --vb-button-background-active: #{var.$background-active};
  --vb-button-foreground: #{var.$foreground};
  --vb-button-foreground-hover: #{var.$foreground-hover};
  --vb-button-foreground-active: #{var.$foreground-active};
  --vb-button-border-color: #{var.$border-color};
  --vb-button-border-color-hover: #{var.$border-color-hover};

### SCSS Variables

<div class="border-top">

  <DataCard 
    id="scss-var-prefix-variable"
    name="$prefix-variable"
    description="Used as the prefix of CSS variables."
    value="core.$prefix-variable"
    computed="vb-"
  />

  <DataCard 
    id="scss-var-prefix-block"
    name="$prefix-block"
    description="Used as the prefix of block names."
    value="core.$prefix-block"
    computed="null"

```scss
$prefix-variable: core.$prefix-variable !default;
$prefix-block: core.$prefix-block !default;
$prefix-element: core.$prefix-element !default;
$prefix-modifier: core.$prefix-modifier !default;
$prefix-modifier-value: core.$prefix-modifier-value !default;

$breakpoints: core.$breakpoints !default;

$transition-property: background, color, border-color, box-shadow !default;
$transition-duration: core.$transition-duration-short !default;
$transition-timing-function: core.$transition-timing-function !default;

$size: core.$form-control-size !default;
$padding: calc(#{list.nth(core.$padding, 1)} - 1px) list.nth(core.$padding, 2) !default;
$gap: 0.5rem !default;
$border-width: 1px !default;
$border-radius: core.$border-radius !default;
$font-size: 1em !default;
$font-weight: inherit !default;
$line-height: core.$line-height !default;

$background: transparent !default;
$background-hover: transparent !default;
$background-active: transparent !default;

$foreground: theme.get("foreground") !default;
$foreground-hover: null !default;
$foreground-active: null !default;

$border-color: theme.get("border-color-dark");
$border-color-hover: theme.get("border-color-darker") !default;
$border-color-active: null !default;

$box-shadow: 0 0 0 0 rgba(black, 0) !default;
$box-shadow-focus: 0 0 0 0.2rem rgba(black, 0.15) !default;

// disabled
$disabled-opacity: 0.6 !default;

// is-loading
$loading-size: 1em !default;
$loading-border: 2px solid !default;
$loading-border-tpl: 1 1 0 0;
$loading-animation-duration: 0.6s !default;
$loading-animation-timing-function: linear !default;

// button_icon
$icon-padding: calc(#{list.nth(core.$padding, 1)} - 1px);
$icon-padding-sm: calc(#{list.nth(core.$padding-sm, 1)} - 1px);
$icon-padding-lg: calc(#{list.nth(core.$padding-lg, 1)} - 1px);

// button_size_sm
$size-sm: core.$form-control-size-sm !default;
$size-sm-padding: calc(#{list.nth(core.$padding-sm, 1)} - 1px) list.nth(core.$padding-sm, 2) !default;
$size-sm-font-size: core.$font-size-sm !default;
$size-sm-line-height: core.$line-height-sm !default;

// button_size_lg
$size-lg: core.$form-control-size-lg !default;
$size-lg-padding: calc(#{list.nth(core.$padding-lg, 1)} - 1px) list.nth(core.$padding-lg, 2) !default;
$size-lg-font-size: core.$font-size-lg !default;
$size-lg-line-height: core.$line-height-lg !default;
```
