## Options

Utility comes with the following Sass configuration options that can be set using either the [config module](/packages/core/config#using-config-module) or [Sass `with` clause](/packages/core/config#using-with-clause) methods.

<CodeExpander lang="scss">
```scss
@use "@vrembem/utility" with (
  $config: (
    // Controls whether module styles are output. This can be overridden on a per 
    // module basis using the `utility-output-exceptions` config option.
    // @type boolean
    "utility-output": true,

    // Modules to exclude from the output rule set in `utility-output`
    // @type list
    "utility-output-exceptions": (),

    // A map containing module names as keys and their preferred class selectors
    // @type map
    "utility-classes": (),

    // A prefix to apply to utility class selectors. Provided value is 
    // automatically suffixed with a "-" character when applied.
    // @type string
    "utility-prefix": "",

    // Whether or not utility styles should be output with the !important flag
    // @type boolean
    "utility-important": false,

    // A map of border-radius utilities to output
    // @type map
    "utility-border-radius": (
      "": tokens.get("border-radius"),
      "lg": tokens.get("border-radius-lg"),
      "circle": tokens.get("border-radius-circle"),
      "square": 0
    ),

    // A list of display value properties to output
    // @type list
    "utility-display": (
      "breakpoints": true,
      "inline": inline,
      "flex": flex,
      "inline-flex": inline-flex,
      "grid": grid,
      "inline-grid": inline-grid,
      "block": block,
      "inline-block": inline-block,
      "none": none
    ),

    // A spacing scale used to output margin, padding, and spacing utilities
    // @type map
    "utility-spacing": (
      "0": 0,

      "sm": calc(tokens.get("spacing") * 2),
      "": calc(tokens.get("spacing") * 4),
```

## Output

Utility modules each have their own output styles. These are all output by default when the utility package is loaded. There are two ways to control which modules are output using the `utility-output` and `utility-output-exceptions` configuration options.

<ReferenceCard ref="config" name="utility-output" type="boolean" defaults="true">
  Controls whether module styles are output. Specify exceptions using the `utility-output-exceptions` config option.

```scss
@include config.set("utility-output", false);
```
</ReferenceCard>

<ReferenceCard ref="config" name="utility-output-exceptions" type="list" defaults="()">
  Modules listed in this configuration are exceptions to the value set in `utility-output`. If `utility-output` is `true`, modules listed will not be output. If `utility-output` is `false`, modules listed will be output.

```scss
@include config.set("utility-output-exceptions", (
  "background", "foreground"
));
```
</ReferenceCard>

#### Example

Use the `utility-output` configuration to toggle output for all modules. Add exceptions to the `utility-output-exceptions` list to override the output option for specific modules.

<CodeExample lang="scss">
```scss
// Import the config module
@use "@vrembem/core/config";

// Disable the output of specific modules
@include config.set("utility-output-exceptions", (
  "gap",
  "margin",
  "padding"
));

// Disable the output of all modules except for "margin" and "padding"
@include config.set((
  "utility-output": false
  "utility-output-exceptions": (
    "margin",
    "padding"
  )
));
```
</CodeExample>

## Basic usage

Once loaded, the utility package provides a number of classes that can be used independently or to supplement other loaded components.

<CodeExample>
  ```html
  <!-- Using utility classes independently -->
  <div class="padding background-shade border border-radius">
    <p class="text-bold text-align-center">...</p>
  </div>

  <!-- Using utility classes with other components -->
  <div class="grid justify-content-center">
    <div class="grid__item row-full">...</div>
    <div class="grid__item row-auto">...</div>
  </div>
  ```
</CodeExample>

To customize utility class selectors used by modules, map the module name to your preferred class selector in the `utility-classes` config option.

<ReferenceCard ref="config" name="utility-classes" type="map" defaults='("separator": "sep")'>
  A map containing module names as keys and their preferred class selectors as values. For example, to use the class `.radius` instead of `.border-radius`, set the following value:

```scss
@include config.set("utility-classes", (
  "border-radius": "radius"
));
```
</ReferenceCard>

It's also possible to provide a prefix to all utility classes using the `utility-prefix` config option.

<ReferenceCard ref="config" name="utility-prefix" type="string" defaults='""'>
  A prefix to apply to utility class selectors. Provided value is automatically suffixed with a `"-"` character when applied.

```scss
@include config.set("utility-prefix", "u");
// Result: .display-none > .u-display-none
```
</ReferenceCard>

## Modules

Below is a list of each utility module, their available variants and intended usage. Some modules also have configuration options that can be used to configure their output.

### background

Utilities for setting the background-color property of an element. Variants are built from both the core `palette` as well as the `palette-variants` configuration options.

export const background = (() => {
  const range = [50, 75, 100, 200, 300, 400, 500, 600, 700, 800, 900, 950];
  const palette = ["primary", "secondary", "neutral", "important"];
  return palette.reduce((acc, pal) => {
    acc[`background-${pal}`] = `background-color: var(--vb-${pal});`
    acc[`background-${pal}-auto`] = `background-color: var(--vb-${pal}-background);`
    range.forEach(val => {
      acc[`background-${pal}-${val}`] = `background-color: var(--vb-${pal}-${val});`;
    });
    return acc;
  }, {});
})();
