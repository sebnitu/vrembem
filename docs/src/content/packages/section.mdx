## Options

Section is built as a Sass module. Sass options are used to control CSS output, while design token CSS variables control the styling of the component. Below are the available configuration options.

### Sass

Section comes with the following Sass configuration options that can be set using either the [config module](/packages/core/config#using-config-module) or [Sass `with` clause](/packages/core/config#using-with-clause) methods.

<CodeExample lang="scss">
```scss
@use "@vrembem/section" with (
  $config: (
    // Map for outputting section_padding_[key] modifier variants
    // @type map
    "section-padding": (
      "breakpoints": true,
      "default": (
        "md": 2em 1.5em,
        "lg": 3em 1.5em
      ),
      "sm": (
        "initial": 1.5em,
      ),
      "lg": (
        "md": 4em 1.5em,
        "lg": 6em 2em
      ),
      "xl": (
        "sm": 3em 1.5em,
        "md": 8em 2em,
        "lg": 12em 4em
      )
    )
  )
);
```
</CodeExample>

### Design tokens

Section uses design token CSS variables in a reference and fallback pattern. Variables are referenced in the component's styles but are not directly defined; instead, they are provided with sensible fallbacks.

  "--vb-section-screen-opacity": "0.8",

## Basic usage

Sections are composed using the base wrapper (`section`) along with three optional elements. The `section__container` element's primary purpose is to wrap content with a max-width and center using margin auto. The other two element's—`section__background` and `section__screen`—add a background image and colored screen that fills the full size of the section component.

- `section`
  - `section__container`
  - `section__background`
  - `section__screen`

<CodeExample flush>
  <Fragment slot="output">
    <section class="section">
      <div class="section__container spacing text-align-center">
        <h1 class="h1">Section example</h1>
        <p>This is a demonstration of the section component using a screen and background image. The screen, background and foreground colors can be adjusted using CSS variables.</p>
        <hr class="sep margin-y-xl" />
        <div class="flex justify-content-center">
          <button class="button button_color_primary">First action</button>
          <button class="button button_color_secondary">Second action</button>
        </div>
      </div>
      <img class="section__background" src="https://picsum.photos/id/335/600/300" />
      <div class="section__screen"></div>
    </section>
  </Fragment>
  ```html
  <section class="section">
    <div class="section__container">
      ...
    </div>
    <img class="section__background" src="..." />
    <div class="section__screen"></div>
  </section>
  ```
</CodeExample>

Section screens and backgrounds are displayed behind the section container. These can be paired with the `vb-theme-light` or `vb-theme-dark` theme classes to create a light or dark section theme.

## section_padding_[key]

A modifier that adjusts the dynamic padding of a section. These modifiers are generated using key/value pairs of the `section-padding` config options map.

<CodeExample flush>
  <Fragment slot="demo">

      <option value="section">None</option>
      <option value="section section_padding_sm">section_padding_sm</option>
      <option value="section section_padding_lg">section_padding_lg</option>
      <option value="section section_padding_xl">section_padding_xl</option>

        <div class="flex justify-content-center">
          <button class="button button_color_primary">First action</button>
