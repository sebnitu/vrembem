## Basic usage

To pass custom configuration to collection entries using CSS custom properties, add the `cssConfig` plugin to your collection instantiation.

<CodeExample lang="js">
```js
import Popover from "@vrembem/popover";
import { cssConfig } from "@vrembem/core";

const popover = new Popover({
  plugins: [
    cssConfig()
  ]
});

await popover.mount();
```
</CodeExample>

When entries are created, `cssConfig` will build a configuration source by getting the values of CSS custom properties based on the keys in the collection's config object.

<CodeExample lang="html">
```html
<div 
  id="custom-popover" 
  class="popover" 
  styles="--vb-popover-event: hover"
>
  ...
</div>
```
</CodeExample>

Properties are prefixed with the components name and Vrembem custom properties prefix if defined. Custom properties are also kebab-case, while their config counterparts are set as camelCase.

<CodeExample lang="js">
```js
// Config option:
popover.config.shiftPadding

// Custom property equivalent:
// --vb-popover-shift-padding
```
</CodeExample>

The prefix is defined under the custom property `--vb-prefix`, which defaults to `vb`.

## Configuration

The following configurations can be used to customize the behavior of the `cssConfig` plugin.

<ReferenceCard name="name" type="string" defaults='"cssConfig"'>
  The unique name of the plugin.
</ReferenceCard>

<ReferenceCard name="sourceKey" type="string" defaults='"attr"'>
  Determines the key under which CSS custom properties derived configuration data is stored in the entry's ConfigManager.
</ReferenceCard>

<ReferenceCard name="updateEvent" type="string" defaults='"updateCustomProps"'>
  The name of the event that triggers an update of the CSS custom properties configuration data.
</ReferenceCard>
