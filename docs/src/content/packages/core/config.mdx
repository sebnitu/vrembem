## Basic usage

The primary purpose of `core/config` is to provide a straightforward API for managing library wide configuration options. You can set, get, remove, and log configuration values that affect all components.

Configuration options are stored in a flat map of key/value pairs. These can be changed using the `config.set` mixin or new configurations can be set as needed. The values that are predefined by core modules are used internally by the entire Vrembem library.

<CodeExample lang="scss">
  ```scss
  $-config: (
    "prefix-block": null,
    "prefix-element": "__",
    "prefix-modifier": "_",
    "prefix-modifier-value": "_",
    "prefix-utility": null,
    "prefix-variable": "vb-",
    "palette-default": 50,
    "palette-range": (0, 5, 10, 12, 15, 20, 30, 40, 50, 60, 70, 80, 90, 95, 98, 100),
    "theme-prefix": "vb-theme-",
    "theme-default": "light",
  );
  ```
</CodeExample>

Values from the configuration map can be used by first importing the configuration module and then calling the `config.get` function with the key of the configuration value to return.

<CodeExample lang="scss" tabs>
  <Fragment slot="tab-scss">
    ```scss
    @use "@vrembem/core/config";

    .box {
      background: #ddd;
      color: #333;
    }

    .#{config.get("theme-prefix")}high-contrast {
      .box {
        background: #fff;
        color: #000;
      }
    }
    ```
  </Fragment>
  <Fragment slot="tab-css">
    ```css
    .box {
      background: #ddd;
      color: #333;
    }

    .vb-theme-high-contrast .box {
      background: #fff;
      color: #000;
    }
    ```
  </Fragment>
</CodeExample>

### Prefixes

Vrembem utilises the BEM naming convention but also allows the ability to adjust the specific syntax using the config module. If you prefer a modifier prefix of `"--"` instead of the default `"_"`, you could do that using `config.set`:

<CodeExample lang="scss">
  ```scss
  // Inside a _setup.scss file
  // Import our configuration module
  @use "@vrembem/core/config";

  // Override the default prefix modifier configuration
  @include config.set("prefix-modifier", "--");
  ```
</CodeExample>

Now all subsequent important Vrembem components would use the `"--"` modifier prefix:

<CodeExample lang="scss">
  ```scss
  // The default block modifier prefix uses "_"
  .button_block {
    // ...
  }

  // If we change the block modifier prefix to "--"
  .button--block {
    // ...
  }
  ```
</CodeExample>

It's important to keep in mind that all changes to the global config should be done in a setup file that is loaded before all other Vrembem components. This ensures that components use the correct configuration values.

<CodeExample lang="scss">
  ```scss
  // Inside an app.scss manifest file
  // This is where me make all our configuration changes
  @use "_setup.scss";

  // Import the entire Vrembem library
  @use "vrembem";
  ```
</CodeExample>

## Reference

This reference details the available configuration API, their arguments, and usage examples so you can customize and control global settings across all Vrembem components.

### config.get()

Returns the value of an option in the configuration map.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$keys...` | string | A key or list of keys to follow in the configuration map and return. |

#### Example

<CodeExample lang="scss">
  ```scss
  @debug config.get("theme-default");
  // Debug: light

  @debug config.get("prefix-variable");
  // Debug: vb-
  ```
</CodeExample>

### config.set()

Set an option in the configuration map in various ways.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$key` | string/map | The key in the map to set. Can be passed a map of key/value pairs. |
| `$value` | any | The value to be stored. Can be passed a map of key/value pairs where key will be prefixed with the provided value keys. |
| `$default (false)` | boolean | Whether or not the value being set is a default value. |

#### Example

<CodeExample lang="scss">
  ```scss
  // Set a single option
  @include config.set("default-theme", "dark");

  // Set multiple options at once
  @include config.set((
    "prefix-variable": "custom-",
    "default-theme": "light"
  ));

  // Set nested options
  @include config.set("prefix", (
    "variables": "custom-",
    "themes": "custom-theme-"
  ));
  ```
</CodeExample>

### config.set-default()

An alias mixin for `config.set()` that predefines the default argument to `true`. Running this will not override a previously existing value.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$key` | string/map | The key in the map to set. Can be passed a map of key/value pairs. |
| `$value` | any | The value to be stored. Can be passed a map of key/value pairs where key will be prefixed with the provided value keys. |

#### Example

<CodeExample lang="scss">
  ```scss
  // Set the default value of the provided key
  @include config.set-default("theme-default", "light");

  // This set call will override the default value of "light"
  @include config.set("theme-default", "dark");

  // This will have no effect since "theme-default" has already been set
  @include config.set-default("theme-default", "high-contrast");

  @debug config.get("theme-default"); // Debug: dark
  ```
</CodeExample>

### config.remove()

Remove an option from the configuration map.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$keys...` | string | The list of keys to follow and remove from the configuration map. |

#### Example

<CodeExample lang="scss">
  ```scss
  // Remove the default theme option
  @include config.remove("default-theme");
  ```
</CodeExample>

### config.log()

Log to console the entire configuration map.

#### Example

<CodeExample lang="scss">
  ```scss
  @include config.log();
  ```
</CodeExample>
