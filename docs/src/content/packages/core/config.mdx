## Basic usage

The primary purpose of `core/config` is to provide a straightforward API for managing library wide configuration options. You can set, get, remove, and log configuration values that affect all components.

### config.get()

Returns the value of an option in the configuration map.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$keys...` | string | A key or list of keys to follow in the configuration map and return. |

#### Example

<CodeExample lang="scss">
  ```scss
  @debug config.get("default-theme");
  // Debug: light

  @debug config.get("prefix-variables");
  // Debug: vb-
  ```
</CodeExample>

### config.set()

Set an option in the configuration map in various ways.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$key` | string/map | The key in the map to set. Can be passed a map of key/value pairs. |
| `$value` | any | The value to be stored. Can be passed a map of key/value pairs where key will be prefixed with the provided value keys. |

#### Example

<CodeExample lang="scss">
  ```scss
  // Set a single option
  @include config.set("default-theme", "dark");

  // Set multiple options at once
  @include config.set((
    "prefix-variables": "custom-",
    "default-theme": "light"
  ));

  // Set nested options
  @include config.set("prefix", (
    "variables": "custom-",
    "themes": "custom-theme-"
  ));
  ```
</CodeExample>

### config.remove()

Remove an option from the configuration map.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$keys...` | string | The list of keys to follow and remove from the configuration map. |

#### Example

<CodeExample lang="scss">
  ```scss
  // Remove the default theme option
  @include config.remove("default-theme");
  ```
</CodeExample>

### config.log()

Log to console the entire configuration map.

#### Example

<CodeExample lang="scss">
  ```scss
  @include config.log();
  ```
</CodeExample>
