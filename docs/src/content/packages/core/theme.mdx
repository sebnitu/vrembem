## Basic usage

The most common use-case for themes is providing a light and dark mode but also any custom theme can be created and used. This module provides mixins for setting, removing, and rendering theme values, supporting dynamic theming and system color scheme preferences.

### theme.set()

Set a new or modify existing theme properties. These are stored as custom properties in the `core/css` module.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$component` | string | The component to set a property value for. |
| `$theme` | string | The theme to set a property value for. |
| `$prop (null)` | string/map | The property value to set for the theme. Can be a map containing prop/value pairs. |
| `$value (null)` | any | The value to set for the respective property. |

#### Example

<CodeExample lang="scss">
  ```scss
  // Set the foreground custom property for the card dark theme.
  @include theme.set("card", "dark", "foreground", blue);

  // Set the background and foreground custom properties for the card light theme using a map of property/value pairs.
  @include theme.set("card", "light", (
    "background": blue,
    "foreground": white,
  ));

  // Create an empty theme by not passing a property or value. This will take the properties of the default theme once output.
  @include theme.set("card", "dark");
  ```
</CodeExample>

### theme.remove()

Remove a custom property from themes in a variety of ways.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$component` | string | The component name to search under. Set to "*" for the following `$theme` and `$prop` arguments to apply to all component themes. |
| `$theme` | string | The component theme to search under or remove. Set to "*" to remove all themes from the specified component or if a `$prop` is provided, all the property from all component themes. |
| `$prop ("*")` | string | The property name to remove. Set to "*" to remove the entire theme from the specified component. |

#### Example

<CodeExample lang="scss">
  ```scss
  // Remove the dark theme from buttons component
  @include theme.remove("button", "dark");

  // Remove the border-color property from all button themes
  @include theme.remove("button", "*", "border-color");

  // Remove the background property from the button light theme
  @include theme.remove("button", "light", "border-color");

  // Remove all themes from all components
  @include theme.remove("*", "*");
  ```
</CodeExample>

### theme.output-theme()

Output all the custom properties and values of a component theme or output a specific theme from all components. The root selector or theme class is not included in the output.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$component ("*")` | string | The component to output. Use "*" or "all" to output a theme from all components. |
| `$theme ("light")` | string | The theme to output. |

#### Example

<CodeExample lang="scss">
  ```scss
  // Output the custom properties for the button light theme.
  .vb-theme-light {
    @include theme.output-theme("button", "light");
  }

  // Output the custom properties for the light theme from all components.
  .vb-theme-light {
    @include theme.output-theme("*", "light");
  }
  ```
</CodeExample>

### theme.output()

Output all the themes for a specific component. Each theme will be wrapped in its own appropriate root selector and theme class. Themes named "light" or "dark" will have a `prefers-color-scheme` media query applied.

#### Arguments

| Name | Type | Description |
|------|------|-------------|
| `$component ("*")` | string | The component to output all applied themes. If no component is passed or set to "*" or "all", will output all applied themes for all components. |

#### Example

<CodeExample lang="scss">
  ```scss
  // Output all themes of all components.
  @include theme.output();

  // Output all themes of the notice component.
  @include theme.output("notice");
  ```
</CodeExample>
