## Basic usage

The most common use case for the `teleport` plugin is when (due to template constraints) modals or popovers are not in an optimal location in the document. Teleporting these elements to a specific location in the document can help improve semantics, accessibility and helps resolve possible style inheritance issues.

To teleport collection entry elements to a specific location in the document, add the `teleport` plugin to your collection instantiation and make sure to set the `where` configuration.

<CodeExample lang="js">
```js
import Modal from "@vrembem/modal";
import { teleport } from "@vrembem/core";

const modals = new Modal({
  plugins: [
    teleport({
      where: ".modals"
    })
  ]
});

await modals.mount();
```
</CodeExample>

## Configuration

The following configurations can be used to customize the behavior of the `teleport` plugin.

<ReferenceCard name="name" type="string" defaults='"teleport"'>
  The unique name of the plugin.
</ReferenceCard>

<ReferenceCard name="where" type="HTMLElement | string" defaults='""'>
  Where the entry element should be teleported. Can either be an HTML element or a valid CSS selector.
</ReferenceCard>

<ReferenceCard name="how" type='"after" | "before" | "append" | "prepend"' defaults='"append"'>
  How the entry element should be teleported relative to the `where` element.
</ReferenceCard>
