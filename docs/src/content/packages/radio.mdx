# Radio

Radios allow the user to select a single option from a set.

```sh
npm install @vrembem/radio
```

<CodeExample lang="scss">
  ```scss
  @use "@vrembem/radio";
  ```
</CodeExample>

## Config options

Radio comes with the following configuration options that can be set using either the [config module](/packages/core/config#using-config-module) or [Sass with clause](/packages/core/config#using-with-clause) methods.

```scss
$config: (
  // Map for outputting radio_size_[key] modifier variants
  // @type map
  "radio-size": (
    "breakpoints": true,
    "default": css.get("form-control-size"),
    "sm": css.get("form-control-size-sm"),
    "md": css.get("form-control-size"),
    "lg": css.get("form-control-size-lg")
  )
);
```

## Basic usage

Radio buttons are composed using a set of `<span>` elements alongside the native `<input type="radio">` element which should be given the `radio__native` class and come before the remaining presentational `<span>` elements.

<CodeExample>
  <Fragment slot="output">
    <span class="radio">
      <input type="radio" class="radio__native" name="example-1" value="1" checked />
      <span class="radio__background">
        <span class="radio__circle">
          <span class="radio__dot"></span>
        </span>
      </span>
    </span>
    <span class="radio">
      <input type="radio" class="radio__native" name="example-1" value="2" />
      <span class="radio__background">
        <span class="radio__circle">
          <span class="radio__dot"></span>
        </span>
      </span>
    </span>
  </Fragment>
  ```html
  <span class="radio">
    <input type="radio" class="radio__native">
    <span class="radio__background">
      <span class="radio__circle">
        <span class="radio__dot"></span>
      </span>
    </span>
  </span>
  ```
</CodeExample>

## radio_size_[key]

            </span>
          </span>

        <span class="radio radio_size_sm">

- `radio_size_sm`
- `radio_size_lg`

### Breakpoint variants

These modifiers come with media breakpoint variants. This allows changing styles based on a specific breakpoint key in the [`breakpoints`](/packages/core#config-options) config options map. Breakpoint variants output can be toggled using the `breakpoints` key in the `radio-size` config options map (default: `true`).

<CodeExample>
  <div slot="output" class="spacing">
    <p>Current view-port width: <code data-vp-width>...</code></p>
    <label class="flex flex_inline flex_gap_sm align-items-center">
      <span class="radio radio_size_sm lg:radio_size_md xl:radio_size_lg">
        <input type="radio" class="radio__native" name="example-5" />
        <span class="radio__background">
          <span class="radio__circle">
            <span class="radio__dot"></span>
          </span>
        </span>
      </span>
      <span>Radio</span>
      <span class="badge lg:display-none">size_sm</span>
      <span class="badge display-none lg:display-block xl:display-none">size_md</span>
      <span class="badge display-none xl:display-block">size_lg</span>
    </label>
  </div>
  ```html
  <span class="radio radio_size_sm lg:radio_size_md xl:radio_size_lg">
    ...
  </span>
  ```
</CodeExample>

**Available variants**

- `xs:radio_size_[key]`
- `sm:radio_size_[key]`
- `md:radio_size_[key]`
- `lg:radio_size_[key]`
- `xl:radio_size_[key]`
